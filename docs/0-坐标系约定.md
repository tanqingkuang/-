# 坐标系与符号约定

## 1. 适用范围

本文档冻结仿真环境、动力学模型、制导/编队算法、日志和 UI 之间共享的坐标与符号契约。其他文档只描述本模块的使用方式；出现冲突时，以本文档为准。

## 2. 地面坐标系：ENU

业务层统一使用右手东北天坐标系 `I=[E,N,U]`：`E` 向东、`N` 向北、`U` 向上。历史字段 `x_m/y_m/altitude_m` 分别表示 `E/N/U`；带方向名的字段使用 `east/north/up`。

Qt Quick 3D 只在显示适配层使用 `[x,y,z]=[E,U,-N]`。该映射不得反向渗透到配置、算法、模型、日志或指标定义。

## 3. 航迹坐标系：FUR

航迹系统一采用右手 `FUR` 轴序：`x` 前向（Forward）、`y` 上法向（Up-normal）、`z` 右侧向（Right）。对航迹倾角 `theta` 和水平航迹角 `psi`，基向量为：

```text
e_x = [ cos(theta) cos(psi),  cos(theta) sin(psi), sin(theta)]^T
e_y = [-sin(theta) cos(psi), -sin(theta) sin(psi), cos(theta)]^T
e_z = [ sin(psi),            -cos(psi),            0         ]^T

C_k^I = [e_x, e_y, e_z]
v_k = (C_k^I)^T v_I
```

业务上存在两个轴序和符号相同、但参考速度不同的航迹系，禁止混用：

- **动力学空速航迹系**：由飞机相对空气的速度方向建立，用于三自由度积分和过载分解。风扰只修正地速/位置推进，不改变该空速航迹系。
- **制导地速航迹系**：由地速或指令地速方向建立，用于航线跟踪、编队槽位、控制误差和诊断量。`track_*` 误差以目标/指令地速建系；目标水平地速不足以定向时可退回本机地速建系。

`FormPosS` 以及队形配置的 `x_forward_y_up_z_right` 均使用完整三维制导 FUR：`x` 前向、`y` 上法向、`z` 右侧向；爬升/下降时，槽位 `x/y` 随参考航迹倾角共同旋转。编队槽位优先以长机指令地速建系，指令水平地速不足以定向时退回长机实际地速，两者都退化时使用东向平飞 FUR 兜底。

集结 `JOINING` 阶段的松散点是一个明确的阶段性投影：它把 `FormPosS` 放入“与任务首段水平航向对齐、`theta=0`”的合成平飞 FUR，生成固定 ENU 水平盘旋几何，并让高度通道独立收敛；它不是第三套轴序，也不是实时三维槽位。进入使用 `SlotGeometry` 的阶段后，槽位仍按上一段的完整三维 FUR 解算。

状态接口分层如下：

- `AircraftState` 的 `speed_mps/theta_rad/psi_rad/psi_dot_deg_s` 是动力学空速航迹状态；`air_v*` 是其 ENU 分量。
- `NodeState`/快照对外的 `vx_mps/vy_mps/vz_mps`、`ground_speed_mps`、`theta_deg/psi_v_deg/psi_dot_deg_s` 是地速航迹状态，供制导、日志和 UI 使用。
- `NodeState.speed_mps` 暂保留为空速兼容字段；`airspeed_mps` 是数值相同但语义显式的别名，`air_theta_deg/air_psi_v_deg/air_psi_dot_deg_s` 保留动力学空速航迹角及角速率，供物理包线裁决。需要地速标量时必须显式使用 `ground_speed_mps`。算法 `vd` 使用水平地速，不等同于三维 `ground_speed_mps`。

## 4. 角度、转弯与滚转符号

- `psi=0` 指东，`psi=90 deg` 指北；从天向下看，逆时针/左转为正。
- 航迹角速率 `psi_dot` 与曲率 `kappa=dpsi/ds` 均为左转正、右转负。
- `phi` 为滚转角，**右倾/右翼下沉为正**，左倾为负。
- 内部计算角度使用 `rad`；配置和对外字段使用带 `_deg` 后缀的度，角速率字段同时标明时间单位，例如 `_deg_s`。同名角度仍须结合上一节的空速/地速接口层解释。

因此右倾产生右向过载和右转：`phi>0`、`nz>0`，但 `psi_dot<0`。不要用 `psi` 的左转正号推导 `phi` 也应左倾为正。

## 5. 过载定义与三自由度方程

过载分量均为无量纲 `g` 倍数，并在动力学空速 FUR 中定义：

- `nx`：前向有符号分量。
- `ny`：上法向有符号分量。
- `nz`：右侧向有符号分量。
- `n_normal=sqrt(ny^2+nz^2)`：法向合过载，恒非负。

滚转角及分量关系为：

```text
phi = atan2(nz, ny)
ny = n_normal cos(phi)
nz = n_normal sin(phi)
n_total = sqrt(nx^2 + ny^2 + nz^2)
        = sqrt(nx^2 + n_normal^2)
```

对应三自由度方程为：

```text
V_dot     = g (nx - sin(theta))
theta_dot = g / V * (ny - cos(theta))
          = g / V * (n_normal cos(phi) - cos(theta))
psi_dot   = -g nz / (V cos(theta))
          = -g n_normal sin(phi) / (V cos(theta))
```

模型限幅配置使用 `nx_min/max`、`n_normal_min/max` 和 `phi_min/max_deg`；不为 `ny/nz` 分量分别设置独立限幅。

## 6. 误差方向

- `pos_err_*`、`vel_err_*`、`track_pos_err_*`、`track_vel_err_*` 均为 `command - actual`。
- `track_*` 是上述误差在制导地速 FUR 中的分量；未进入当前控制律的诊断通道可显式置 `0`。
- `cross_track_error_m` 的参考对象不是指令点，而是当前航段：飞机位于航段右侧为正、左侧为负。
- 状态表为满足飞行诊断习惯，可以只在显示层对部分 `track_*` 字段取反，展示 `actual - target`；不得修改快照和日志中的原始误差语义。
