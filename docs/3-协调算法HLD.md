# 协调算法 HLD

## 1. 定位

协调算法属于验证对象组，是集中式或参考型策略中的可选算法实例。不同编队方法下，协调算法可以不存在，也可以运行在长机、地面站、虚拟节点或参考节点。

## 2. 职责

- 在需要集中协调的编队方法中生成全局规划、队形指令或参考信号。
- 声明自身需要发送 / 接收的消息 schema。
- 基于仿真控制注入的状态、Inbox 和配置输出协调结果。
- 保持无 I/O、不持仿真环境引用，便于后续迁移到 C 仿真 / 半物理仿真。

## 3. 边界

- 可选实例，纯分布式方法下不创建。
- 不直接访问模型迭代、通信功能、加扰或日志。
- 不直接读取 yaml 配置文件。
- 不理解通信 QoS，只生成 algorithm payload 和 Outbox。

## 4. 主要接口类别

- 生命周期：`init`、`close`
- 算法主接口：`step`
- 消息输入：`deliver_inbox`
- 消息输出：`read_outbox`
- schema 声明：`declare_message_schema`
