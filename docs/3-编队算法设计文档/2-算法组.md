# 一、说明

本文档描述编队算法“算法组”各单元的低层设计（LLD），供人阅读，也用于指导代码开发。

算法组属于 HLD 四层中的 L2，特点是**不感知 Mode（编队模态），只做纯计算**。每个单元是一个策略族：一个抽象类加若干子类。使用者（对象组）只选择子类、绑定输入输出，不关心子类内部实现。

约定：

- 接口与命名遵循《0-HLD.md》2.6；算法组为 L2，接口为 `init/step/reset`（无 `close`），`step` 的输出 `y` 原地写出、无返回值。
- 数据通过黑板 `Context` 交换。输入/输出结构体的字段是指向 `Context` 子对象的“端口”，运行期由对象组绑定到 `Context` 的具体字段（整体规则见《0-HLD.md》3.2、3.3）。本文每个端口都注明它绑定到 `Context` 的哪个字段。
- 叶类型（结构体、枚举）定义见《1-Context和叶类型.md》，本文直接引用。
- 坐标系：`Context` 与各单元对外 I/O 统一用东北天（east/north/h）。单元内部计算如需其它坐标系（如苏联式本机航迹系，用于在固定翼约束下区分前向/法向/侧向右，见 PosTrack），由 `FormationMath` 转换，转换只发生在单元内部，不改变对外的东北天约定。
- 本轮覆盖用例：1 个航段 + 集中式通信 + 长机僚机 + PID 控制 + 三角队形（结果报告见《5-用例-领航跟随保持.md》）。

算法组单元清单：

| 单元 | 名称(XXX) | 作用 |
|---|---|---|
| 位置解算 | PosCalc | 算出本机要跟踪的目标位置与目标速度 |
| 轨迹/编队跟踪 | PosTrack | 由目标与当前状态直接算出三轴加速度指令（内部完成求偏差与坐标变换） |
| 控制算法 | Ctrl | 单轴原子控制律（PID 等），被 PosTrack 组合调用 |
| 通用数学 | FormationMath | 坐标变换、限幅、滤波等工具函数（非策略族） |

> 说明：早期单独有一个“误差解算”单元，先算误差再交给跟踪。现已把求误差并入 PosTrack——误差只是中间量，没必要作为单独工序、也没必要进 Context。

# 二、各算法

## 2.1 位置解算 PosCalc

作用：算出本机当前要跟踪的目标运动状态（目标位置 + 目标速度），写入 `Context.selfCmd`。长机按航段插值得到目标，僚机按队形槽位相对长机得到目标。

### 2.1.1 抽象类

接口：

```python
class PosCalcBase:
    def init(self, cfg: "PosCalcInitS") -> None: ...
    def step(self, u: "PosCalcInputS", y: "PosCalcOutputS") -> None: ...  # y 原地写出
    def reset(self) -> None: ...
```

公共输入/输出结构体（端口）：

```python
@dataclass
class PosCalcInitS:          # 公共初始化配置（子类按需扩展）
    pass

@dataclass
class PosCalcInputS:         # 公共输入端口
    selfState: MotionProfS = None   # 端口 → Context.selfState，本体运动状态

@dataclass
class PosCalcOutputS:        # 公共输出端口
    selfCmd: MotionProfS = None     # 端口 → Context.selfCmd，本机目标运动状态
```

### 2.1.2 子类：航线插值 RouteInterp（长机用）

实现方法：在当前跟踪航段上，把本体位置投影到航段；若配置了 `lookAheadDistance`，再沿航段方向前移该距离，得到目标位置（东北天表示）。目标速度大小取航段速度指令 `vdCmd`，方向沿航段。当前只支持直线段（`radius==0`）；曲线段接口预留、暂不实现。

补充：目标位置取航段上的投影点或前视点，沿航段前向不设独立的位置闭环——前向由速度指令 `vdCmd` 控制；前向、法向和侧向右的区分由 PosTrack 在苏联式本机航迹系下处理（见 2.2）。

结构体（在公共端口上扩展）：

```python
@dataclass
class RouteInterpInitS(PosCalcInitS):
    lookAheadDistance: float = 0.0  # 航段前视距离；0 表示只使用投影点

@dataclass
class RouteInterpInputS(PosCalcInputS):
    wayLine: WayLineS = None    # 端口 → Context.wayLine，当前跟踪航段
# 输出沿用 PosCalcOutputS（selfCmd）
```

接口：实现 `init/step/reset`。`step` 读 `u.wayLine` 与 `u.selfState`，写 `y.selfCmd`。

测试用例设计：

- 直线航段 + 本体在航段起点：目标位置在航段上，目标速度方向沿航段、大小等于 `vdCmd`。
- 本体偏离航段：目标位置为本体在航段上的投影点。
- 配置前视距离：目标位置应从投影点沿航段方向前移指定距离。
- `radius != 0`（曲线段）：当前明确不支持，应给出清晰报错或标志，用于回归保护。

### 2.1.3 子类：槽位几何 SlotGeometry（僚机用）

实现方法：用当前队形 `cmd.pattern` 在 `self.formPat` 中反查到对应的行号 `m`（`m = formPat.index(cmd.pattern)`），再在 `formPos[m]` 中找出 `id == selfId` 的那个 `FormPosS`，得到本机槽位相对长机的偏移；目标位置 = 长机位置 + 槽位偏移，目标速度 = 长机速度 + 槽位误差反馈修正。队形几何来自配置文件 `formation.pattern` / `formation.slots`，在 `init` 时由对象组注入，存于 `self`。

说明：行号 `m` 必须经 `formPat` 反查，不能直接拿枚举值 `cmd.pattern` 当下标——`formPos` 只包含实际配置的队形（可能只有一行），枚举值与行号不一定一致，直接当下标会错位或越界。本机槽位也按 `FormPosS.id == selfId` 匹配（id 为字符串节点 ID），不按数组位置取。

结构体：

```python
@dataclass
class SlotGeometryInitS(PosCalcInitS):
    selfId: str = ""                                            # 本机 ID（节点ID，如 "A01"）
    formPat: list[FormPatE] = field(default_factory=list)        # [M] 第 m 行对应的队形，用于把 pattern 反查成行号
    formPos: list[list[FormPosS]] = field(default_factory=list)  # [M][N] 各队形各槽位相对位置

@dataclass
class SlotGeometryInputS(PosCalcInputS):
    leaderState: MotionProfS = None   # 端口 → Context.leaderState，长机运动状态
    cmd: FormSnapshotS = None         # 端口 → Context.cmd，取当前队形 pattern
# 输出沿用 PosCalcOutputS（selfCmd）
```

接口：实现 `init/step/reset`。`step` 用 `u.cmd.pattern` 经 `self.formPat` 反查行号 `m`，在 `self.formPos[m]` 中按 `id==selfId` 找到本机槽位偏移，结合 `u.leaderState` 算出 `y.selfCmd`。

测试用例设计：

- 三角队形 + 给定长机位置：目标位置 = 长机位置 + 该槽位偏移（验证坐标变换正确）。
- 目标速度 = 长机速度 + 槽位误差反馈修正。
- 切换 `pattern`：偏移随队形变化。

### 2.1.N 整理

| 子类 | 接口 | 输入结构体（叶类型） | 输出结构体（叶类型） | 支持的场景与功能 |
|---|---|---|---|---|
| RouteInterp 航线插值 | init/step/reset | RouteInterpInputS：selfState(MotionProfS)、wayLine(WayLineS) | PosCalcOutputS：selfCmd(MotionProfS) | 长机沿航段插值目标；支持可选航段前视距离；仅直线段，曲线段预留 |
| SlotGeometry 槽位几何 | init/step/reset | 输入 SlotGeometryInputS：selfState(MotionProfS)、leaderState(MotionProfS)、cmd(FormSnapshotS)；配置 SlotGeometryInitS：selfId、formPat(list[FormPatE])、formPos(list[list[FormPosS]]) | PosCalcOutputS：selfCmd(MotionProfS) | 僚机按队形槽位跟随长机；pattern 经 formPat 反查行号；支持三角队形 |

## 2.2 轨迹/编队跟踪 PosTrack

作用：根据目标运动状态和本机当前运动状态，直接算出三轴加速度指令，写入 `Context.selfAccCmd`。本单元内部完成“求偏差 → 坐标变换 → 控制律”，不把中间的误差摆到 `Context` 上。

> 设计说明：把求偏差并入本单元有两个原因——① 误差只是中间量，没必要进黑板；② 固定翼控制要在苏联式本机航迹系下做（前向控速度、法向控高度爬升、侧向右控转弯），需要先把东北天的偏差转到航迹系；而本单元同时拿到目标与当前状态，转换最自然。

### 2.2.1 抽象类

```python
class PosTrackBase:
    def init(self, cfg: "PosTrackInitS") -> None: ...
    def step(self, u: "PosTrackInputS", y: "PosTrackOutputS") -> None: ...
    def reset(self) -> None: ...

@dataclass
class PosTrackInitS:
    pass

@dataclass
class PosTrackInputS:
    selfCmd: MotionProfS = None    # 端口 → Context.selfCmd，本机目标运动状态
    selfState: MotionProfS = None  # 端口 → Context.selfState，本机当前运动状态

@dataclass
class PosTrackOutputS:
    accCmd: AccInEarthS = None      # 端口 → Context.selfAccCmd，三轴加速度指令（东北天）
```

### 2.2.2 子类：PID 组合 PidCompose

实现方法（一拍内）：

1. 速度下限检查：读当前地速大小 `u.selfState.v.vd`，若 `< self.vMin`（当前 0.5 m/s）则**立即报错并中止**，不继续计算（见下方说明）。
2. 求位置偏差：位置偏差 = 目标位置 − 当前位置（东北天）。
3. 转坐标系：用当前状态的速度方向，把位置偏差、当前速度 `u.selfState.v`、目标速度 `u.selfCmd.v` 分别从东北天转到苏联式本机航迹系（前向/法向/侧向右），转换调用 `FormationMath`。
4. 求速度偏差：前向速度误差使用地速标量 `u.selfCmd.v.vd - u.selfState.v.vd`；法向、侧向右速度误差在苏联式本机航迹系内计算 `目标速度分量 − 当前速度分量`，避免把法向、侧向右轴的控制语义混在东北天分量上。
5. 逐轴控制：前向、法向、侧向右各持有一个 `Ctrl`(PID)。前向只做速度 PI，调用方式为 `forward.step(地速误差, 0.0)`，即地速误差走 `kp/ki`，前向位置误差不参与，`kd` 不用于前向速度误差；法向、侧向右用位置偏差 + 速度偏差控制。
6. 转回东北天：把三轴加速度从航迹系转回东北天，写入 `y.accCmd`。

PID 增益和控制周期 `dt` 在 `init` 配置、存于 `self`。其中 `dt` 不由 PID 自行推断，也不使用固定常量；在仿真运行路径中由仿真控制按 `step_s * algorithm_decimation` 计算后，经 `EntityInitS.control_period_s` 注入实体，再由实体写入三轴 `CtrlInitS.dt`。默认配置下 `step_s=0.005`、`algorithm_decimation=10`，因此默认 PID 控制周期为 `0.05 s`。

> **速度下限说明**：航迹系由地速方向定义，地速→0 时方向无定义、坐标变换奇异。但固定翼必须保持空速飞行，巡航中正常不会低于 `vMin`；因此本单元**不实现低速退化**，而是把 `vd < vMin` 当作非法状态（上游软件 bug）直接报错中止（fail-fast），便于尽早暴露问题。`vMin` 当前配置为 0.5 m/s。

结构体：

```python
@dataclass
class PidComposeInitS(PosTrackInitS):
    vMin: float = 0.5                 # 地速下限（m/s）；低于则报错（fail-fast，非退化）
    gainForward: "CtrlInitS" = None   # 前向 PID 配置（本机航迹系）
    gainLateral: "CtrlInitS" = None   # 侧向 PID 配置
    gainVertical: "CtrlInitS" = None  # 高度 PID 配置
# 输入/输出沿用 PosTrackInputS / PosTrackOutputS
```

接口：实现 `init/step/reset`。`init` 内实例化三个 `Ctrl` 并各自 `init`；`step` 按上面步骤算出 `y.accCmd`（地速低于 `vMin` 时报错）；`reset` 内逐轴调用 `Ctrl.reset`。

> **依赖说明**：`PosTrack` 在内部组合调用 `Ctrl`，即一个算法单元依赖另一个算法单元。这符合《0-HLD.md》5.2 的规则——算法单元之间允许最小化、无循环的依赖。`Ctrl` 是被复用的单轴原子控制律，依赖方向单一（`PosTrack → Ctrl`），不构成循环。

测试用例设计：

- 给定目标与当前状态：偏差、坐标变换、加速度结果可用简单几何核对。
- 前向：地速标量误差由 P/I 响应，前向位置偏差不产生前向加速度，`kd` 不响应前向速度误差。
- 法向/侧向右：位置偏差 + 速度偏差按 PID 响应，其中 P 响应位置偏差、I 累积位置偏差、D 响应速度偏差。
- 输出限幅生效；`reset` 后积分清零。
- 地速 < `vMin`：`step` 报错中止，不产生加速度。

|---|---|---|---|---|
| PidCompose PID 组合 | init/step/reset | 输入 PosTrackInputS：selfCmd(MotionProfS)、selfState(MotionProfS)；配置 PidComposeInitS：vMin、前向/法向/侧向右 CtrlInitS | PosTrackOutputS：accCmd(AccInEarthS) | 苏联式本机航迹系下三轴 PID；前向控地速、法向/侧向右控位置+速度；地速<vMin 报错；组合调用 Ctrl |

## 2.3 控制算法 Ctrl

作用：单轴原子控制律。被 `PosTrack` 在内部组合调用，**不直接绑定 `Context`**，输入输出为标量（不受端口绑定粒度限制，因为它不参与 L1 串联）。

### 2.3.1 抽象类

```python
class CtrlBase:
    def init(self, cfg: "CtrlInitS") -> None: ...
    def step(self, posErr: float, velErr: float) -> float: ...  # 返回该轴控制量（加速度分量）
    def reset(self) -> None: ...

@dataclass
class CtrlInitS:
    kp: float = 0.0
    ki: float = 0.0
    kd: float = 0.0
    dt: float = 0.0        # 控制周期，单位 s
    iMax: float = 0.0      # 积分限幅（0 表示不限）
    outMax: float = 0.0    # 输出限幅（0 表示不限）
```

`dt` 必须等于本控制律实际被调用的控制周期。播放倍率和 UI 刷新率不改变 `dt`；修改 `step_s` 或 `algorithm_decimation` 会改变 `dt`，并会影响积分项累积速度和闭环动态。

### 2.3.2 子类：PID

实现方法：`输出 = kp*posErr + ki*积分(posErr) + kd*velErr`。其中 `velErr` 直接作为微分项（速度误差即位置误差的导数），避免对误差做数值差分。积分项做积分限幅，最终输出做输出限幅。积分值与上一拍量存于 `self`。

接口：实现 `init/step/reset`。`reset` 清零积分。

测试用例设计：

- 纯 P（ki=kd=0）：输出 = kp*posErr。
- 纯 I：多拍累积，验证积分增长与限幅。
- 纯 D：输出 = kd*velErr。
- 输出限幅：大误差时输出被限到 `outMax`。
- `reset`：积分清零。

### 2.3.3 预埋子类

- L1（pid.py 同目录预留）：自适应 L1 控制律，本轮不实现。
- ADRC：自抗扰控制律，本轮不实现。

### 2.3.N 整理

| 子类 | 接口 | 输入/输出 | 支持的场景与功能 |
|---|---|---|---|
| PID | init/step/reset | 入：posErr、velErr(float)；出：float | 单轴 PID，带积分限幅与输出限幅（已实现） |
| L1 | 同上 | 同上 | 预埋，未实现 |
| ADRC | 同上 | 同上 | 预埋，未实现 |

## 2.4 通用数学 FormationMath

作用：提供纯工具函数。**不是策略族**，不持内部状态，不走 `init/step/reset` 接口，也不绑定 `Context`；由其它单元直接调用。因此本节不设抽象类/子类，也没有 `2.x.N 整理`表（整理表用于对比策略族的各子类，对纯函数模块不适用）。

函数清单（本轮）：

- 坐标变换：东北天与苏联式本机航迹系之间的向量变换（PosTrack 用）。
- 限幅：`clamp(x, lo, hi)`。
- 滤波：一阶低通等（按需补充）。

测试用例设计：逐个函数做单元测试（给定输入、核对输出与边界）。
