# 一、说明

本文档描述编队算法“算法组”各单元的低层设计（LLD），供人阅读，也用于指导代码开发。

算法组属于 HLD 四层中的 L2，特点是**不感知 Mode（编队模态），只做纯计算**。每个单元是一个策略族：一个抽象类加若干子类。使用者（对象组）只选择子类、绑定输入输出，不关心子类内部实现。

约定：

- 接口与命名遵循《0-HLD.md》2.6；算法组为 L2，接口为 `init/step/reset`（无 `close`），`step` 的输出 `y` 原地写出、无返回值。
- 数据通过黑板 `Context` 交换。输入/输出结构体的字段是指向 `Context` 子对象的“端口”，运行期由对象组绑定到 `Context` 的具体字段（整体规则见《0-HLD.md》3.2、3.3）。本文每个端口都注明它绑定到 `Context` 的哪个字段。
- 叶类型（结构体、枚举）定义见《1-Context和叶类型.md》，本文直接引用。
- 坐标系命名：东北天坐标系字段统一使用 `east/north/h`（位置）和 `east/north/up`（速度、加速度）；航迹坐标系字段统一使用 `x/y/z`，其中 `x` 为前向、`y` 为垂向、`z` 为右侧向。业务控制口默认使用东北天；若为了日志/UI 暴露航迹系诊断量，必须以 `diag` 诊断输出显式带出，不写入 `Context`。
- 覆盖用例：1 个航段 + 集中式通信 + 长机僚机 + PID 控制 + 三角队形（结果报告见《5-用例-领航跟随保持.md》）。

算法组单元清单：

| 单元 | 名称(XXX) | 作用 |
|---|---|---|
| 位置解算 | PosCalc | 算出本机要跟踪的目标位置与目标速度 |
| 轨迹/编队跟踪 | PosTrack | 由目标与当前状态直接算出三轴加速度指令（内部完成求偏差与坐标变换） |
| 控制算法 | Ctrl | 单轴原子控制律（PID 等），被 PosTrack 组合调用 |
| 通用数学 | FormationMath | 坐标变换、限幅、滤波等工具函数（非策略族） |

> 说明：位置/速度误差由 PosTrack 内部计算，不作为独立单元进入 Context。若仿真日志或 UI 需要查看误差，由 PosTrack 在 `y.diag` 中输出本拍诊断量，对象组再汇总到实体边界输出。

# 二、各算法

## 2.1 位置解算 PosCalc

作用：算出本机当前要跟踪的目标运动状态（目标位置 + 目标速度），写入 `Context.selfCmd`。长机按航段插值得到目标，僚机按队形槽位相对长机得到目标。

### 2.1.1 抽象类

接口：

```python
class PosCalcBase:
    def init(self, cfg: "PosCalcInitS") -> None: ...
    def step(self, u: "PosCalcInputS", y: "PosCalcOutputS") -> None: ...  # y 原地写出
    def reset(self) -> None: ...
```

公共输入/输出结构体（端口）：

```python
@dataclass
class PosCalcInitS:          # 公共初始化配置（子类按需扩展）
    pass

@dataclass
class PosCalcInputS:         # 公共输入端口
    selfState: MotionProfS = None   # 端口 → Context.selfState，本体运动状态

@dataclass
class PosCalcOutputS:        # 公共输出端口
    selfCmd: MotionProfS = None     # 端口 → Context.selfCmd，本机目标运动状态
```

### 2.1.2 子类：航线插值 RouteInterp（长机用）

实现方法：按当前航段类型分两支。
- **直线段（`radius==0`）**：把本体位置投影到航段；若配置了 `lookAheadDistance`(L1)，再沿航段方向前移该距离，得到目标位置（东北天）。目标速度大小取 `vdCmd`、方向沿航段。
- **圆弧段（`radius>0`）**：把本体**投影到圆弧**，目标位置取**投影点**（不沿弧前移 L1，避免前视点弓高被横向环当误差→与曲率前馈双算/打架，见 2.2.2）。目标速度沿该处**切向**、大小 `vdCmd`，垂向按弧首末高度沿弧长线性。圆弧的圆心/转向由航路规划在切点几何里算好（见《3-流程组》LeaderRoute）。

补充：直线/圆弧的目标位置都不设独立的前向位置闭环——前向由 `vdCmd` 控制；前向、法向、侧向右的区分由 PosTrack 在苏联式本机航迹系下处理（见 2.2）。

转弯前馈（σ 前瞻曲率）：另写出目标航迹偏航角速率 `selfCmd.v.dVPsi = vd × κ_ff`，供 PosTrack 在侧向轴做向心加速度前馈（见 2.2.2）。其中 `κ_ff` 取**前瞻窗内平均曲率 = (前瞻航向 − 当前航向)/窗长**，窗长 `L2 = σ·vd`（σ 为前瞻时间，长机默认 0.5s，定义在 leader 实体里、紧邻 L1，便于调参）。
- 前瞻点 `s0 + L2` 落在当前段内则在本段取航向；落到段末之外**且下一段是圆弧**时才跨段到 `nextWayLine` 取航向（故需输入 `nextWayLine`）；下一段是直线（含尖角）则不跨段，避免在直线段凭空产生前馈。
- 直线→圆弧时，进弧前 L2 内 `κ_ff` 从 0 平滑渐升到 `1/R`；圆弧→直线时渐降回 0。直线→直线（`R=0` 路线）`κ_ff≡0`。

结构体（在公共端口上扩展）：

```python
@dataclass
class RouteInterpInitS(PosCalcInitS):
    lookAheadDistance: float = 0.0  # 直线 L1 前视距离；0 表示只用投影点
    leadTimeS: float = 0.0          # 曲率前馈前瞻时间 σ(秒)，窗长 L2=σ·vd；0 关闭前馈

@dataclass
class RouteInterpInputS(PosCalcInputS):
    wayLine: WayLineS = None        # 端口 → Context.wayLine，当前跟踪航段
    nextWayLine: WayLineS = None    # 端口 → Context.nextWayLine，供曲率前馈跨段前瞻(仅跨入圆弧)
# 输出沿用 PosCalcOutputS（selfCmd）
```

接口：实现 `init/step/reset`。`step` 读 `u.wayLine` 与 `u.selfState`，写 `y.selfCmd`。

测试用例设计：

- 直线航段 + 本体在航段起点：目标位置在航段上，目标速度方向沿航段、大小等于 `vdCmd`。
- 本体偏离航段：目标位置为本体在航段上的投影点。
- 配置前视距离：目标位置应从投影点沿航段方向前移指定距离。
- 圆弧航段 + 本体在弧上：目标位置=弧上投影点、速度沿切向、`dVPsi = vd·κ`（右转为负）。
- 直线航段（`radius==0`）不产生曲率前馈；直线→圆弧交接前，`nextWayLine` 只在下一段为圆弧时参与 σ 前瞻。

### 2.1.3 子类：槽位几何 SlotGeometry（僚机用）

实现方法：用当前队形 `cmd.pattern` 在 `self.formPat` 中反查到对应的行号 `m`（`m = formPat.index(cmd.pattern)`），再在 `formPos[m]` 中找出 `id == selfId` 的那个 `FormPosS`，得到本机槽位相对长机的偏移；目标位置 = 长机位置 + 槽位偏移，目标速度 = 槽位随转速度前馈 `v_S`（见下方"转弯前馈"）。本单元只产出纯几何目标与速度前馈，**不读本机状态、不做前向待飞距闭环**——前向待飞距收敛已下沉到 PosTrack 的前向位置环（见 2.2.2）。队形几何来自配置文件 `formation.pattern` / `formation.slots`，在 `init` 时由对象组注入，存于 `self`。

说明：行号 `m` 必须经 `formPat` 反查，不能直接拿枚举值 `cmd.pattern` 当下标——`formPos` 只包含实际配置的队形（可能只有一行），枚举值与行号不一定一致，直接当下标会错位或越界。本机槽位也按 `FormPosS.id == selfId` 匹配（id 为字符串节点 ID），不按数组位置取。

转弯前馈分两部分，二者用同一个长机偏航角速率 `ω = leaderState.v.dVPsi`（由长机广播携带、僚机入站解析）：

1. **航迹偏航角速率前馈**（向心加速度）：目标 `selfCmd.v.dVPsi = ω`（恒等）。长机转弯时槽位随其水平航迹刚性旋转，刚体绕同一瞬心、各点航向角速率相同，故僚机目标偏航角速率即长机的；向心加速度 `vd·dVPsi` 由 PosTrack 在侧向轴前馈（见 2.2.2），外/内侧半径差异由本机自身地速 `vd` 自动吸收（`v_S/R_S = dVPsi`）。
2. **槽位速度前馈**（沿航迹快慢）：目标速度取槽位随转的真实速度 `v_S = (V + b·ω)·t̂ + (a·ω)·n̂`（`V` 长机地速、`a=slot.x` 前向偏移、`b=slot.z` 右侧向偏移、`t̂` 长机水平航迹、`n̂` 左单位向量）。沿航迹分量按 `b·ω` 增减：对某一确定转向，半径大的外侧槽位加速、内侧减速，而某个槽位（固定 `b`）落在外侧还是内侧由 `b` 与 `ω` 的符号共同决定（例：左转 `ω>0` 时右槽 `b>0` 为外侧、加速）；`a·ω` 补后方槽位转弯时的横扫。`ω=0`（直线）时 `v_S` 退化为长机速度，行为不变。

> 说明：槽位速度前馈承担转弯时内外侧线速度差的补偿，是纯前馈量；前向待飞距的闭环修正不在本单元做，统一由 PosTrack 的前向位置环按位置误差闭环（见 2.2.2），避免几何单元与控制律耦合。

结构体：

```python
@dataclass
class SlotGeometryInitS(PosCalcInitS):
    selfId: str = ""                                            # 本机 ID（节点ID，如 "A01"）
    formPat: list[FormPatE] = field(default_factory=list)        # [M] 第 m 行对应的队形，用于把 pattern 反查成行号
    formPos: list[list[FormPosS]] = field(default_factory=list)  # [M][N] 各队形各槽位相对位置

@dataclass
class SlotGeometryInputS(PosCalcInputS):
    leaderState: MotionProfS = None   # 端口 → Context.leaderState，长机运动状态
    cmd: FormSnapshotS = None         # 端口 → Context.cmd，取当前队形 pattern
# 输出沿用 PosCalcOutputS（selfCmd）
```

接口：实现 `init/step/reset`。`step` 用 `u.cmd.pattern` 经 `self.formPat` 反查行号 `m`，在 `self.formPos[m]` 中按 `id==selfId` 找到本机槽位偏移，结合 `u.leaderState` 算出 `y.selfCmd`。

测试用例设计：

- 三角队形 + 给定长机位置：目标位置 = 长机位置 + 该槽位偏移（验证坐标变换正确）。
- 目标速度 = 槽位随转速度前馈 `v_S`（直线段退化为长机速度），落后于前向槽位时不叠加追赶速度。
- 切换 `pattern`：偏移随队形变化。

## 2.2 轨迹/编队跟踪 PosTrack

作用：根据目标运动状态和本机当前运动状态，直接算出三轴加速度指令，写入 `Context.selfAccCmd`。本单元内部完成“求偏差 → 坐标变换 → 控制律”，不把中间的误差摆到 `Context` 上；需要日志/UI 查看时，通过 `PosTrackOutputS.diag` 输出本拍诊断副本。

> 设计说明：把求偏差并入本单元有两个原因——① 误差只是中间量，没必要进黑板；② 固定翼控制要在航迹坐标系下做（`x` 前向控速度、`y` 垂向控高度爬升、`z` 右侧向控转弯），需要先把东北天的偏差转到航迹系；而本单元同时拿到目标与当前状态，转换最自然。

### 2.2.1 抽象类

```python
class PosTrackBase:
    def init(self, cfg: "PosTrackInitS") -> None: ...
    def step(self, u: "PosTrackInputS", y: "PosTrackOutputS") -> None: ...
    def reset(self) -> None: ...

@dataclass
class PosTrackInitS:
    pass

@dataclass
class PosTrackInputS:
    selfCmd: MotionProfS = None    # 端口 → Context.selfCmd，本机目标运动状态
    selfState: MotionProfS = None  # 端口 → Context.selfState，本机当前运动状态

@dataclass
class PosTrackOutputS:
    accCmd: AccInEarthS = None      # 端口 → Context.selfAccCmd，三轴加速度指令（东北天）
    diag: PosTrackDiagS = None      # 诊断输出 → 对象组持有，不进 Context，字段见《1-Context和叶类型.md》
```

### 2.2.2 子类：PID 组合 PidCompose

实现方法（一拍内）：

1. 速度下限检查：读当前地速大小 `u.selfState.v.vd`，若 `< self.vMin`（默认 0.5 m/s）则**立即报错并中止**，不继续计算（见下方说明）。
2. 求位置偏差：位置偏差 = 目标位置 − 当前位置（东北天）。
3. 转坐标系：用当前状态的速度方向，把位置偏差、当前速度 `u.selfState.v`、目标速度 `u.selfCmd.v` 分别从东北天转到航迹坐标系（`x` 前向、`y` 垂向、`z` 右侧向），转换调用 `FormationMath`。
4. 求速度偏差：前向速度误差使用地速标量 `u.selfCmd.v.vd - u.selfState.v.vd`；垂向、右侧向速度误差在航迹坐标系内计算 `目标速度分量 − 当前速度分量`，避免把垂向、右侧向轴的控制语义混在东北天分量上。
5. 逐轴控制：前向、垂向、右侧向各持有一个 `Ctrl`(PID)，三轴统一调用 `step(位置误差, 速度误差)`。前向是速度环还是位置环由增益决定，本身不写死：长机型配置 `kp=ki=0`，前向为速度环（速度比例走 `kd`；`kiv` 速度积分预留，当前默认 `kiv=0` 即纯速度 P），前向位置误差被忽略；僚机型配置 `kiv=0`，前向为位置环（待飞距走 `kp/ki`、`kd` 作速度阻尼）。垂向、右侧向恒为位置环，用位置偏差 + 速度偏差控制。
6. 转弯向心前馈：在右侧向轴上叠加前馈 `lateral_ff = -selfCmd.v.dVPsi × selfState.v.vd`，即维持转弯所需的向心加速度 `vd·dVPsi`，作为纯前馈与右侧向 PID 输出相加。取负号是因为航迹坐标系 `z` 右侧向以右为正，而 `dVPsi>0` 为左转；配合本机自身 `vd`，长机圆弧与僚机随转、内外侧僚机的半径/速度差异都被自动吸收。前馈而非反馈：稳态盘旋时 `dVPsi` 非零、向心加速度恒在，避免靠位置误差拉出稳态滞后造成超调。
7. 转回东北天：把三轴加速度从航迹系转回东北天，写入 `y.accCmd`。
8. 写诊断量：若 `y.diag` 已绑定，则把目标位置/速度、东北天误差、航迹系误差写入 `y.diag`；航迹系误差按控制参数屏蔽，某轴位置或速度误差未进入控制律时对应诊断值输出 0；诊断量只供对象组汇总到实体输出，不反向参与控制闭环。

PID 增益和控制周期 `dt` 在 `init` 配置、存于 `self`。其中 `dt` 不由 PID 自行推断，也不使用固定常量；在仿真运行路径中由仿真控制按 `step_s * algorithm_decimation` 计算后，经 `EntityInitS.control_period_s` 注入实体，再由实体写入三轴 `CtrlInitS.dt`。默认配置下 `step_s=0.005`、`algorithm_decimation=10`，因此默认 PID 控制周期为 `0.05 s`。

> **速度下限说明**：航迹系由地速方向定义，地速→0 时方向无定义、坐标变换奇异。固定翼必须保持空速飞行，巡航中正常不会低于 `vMin`；因此 `vd < vMin` 属于非法状态，`step` 直接报错中止。`vMin` 默认配置为 0.5 m/s。

结构体：

```python
@dataclass
class PidComposeInitS(PosTrackInitS):
    vMin: float = 0.5                 # 地速下限（m/s）；低于则快速报错，避免退化
    gainForward: "CtrlInitS" = None   # 前向 PID 配置（本机航迹系）
    gainLateral: "CtrlInitS" = None   # 侧向 PID 配置
    gainVertical: "CtrlInitS" = None  # 高度 PID 配置
# 输入/输出沿用 PosTrackInputS / PosTrackOutputS
```

接口：实现 `init/step/reset`。`init` 内实例化三个 `Ctrl` 并各自 `init`；`step` 按上面步骤算出 `y.accCmd`（地速低于 `vMin` 时报错）；`reset` 内逐轴调用 `Ctrl.reset`。

> **依赖说明**：`PosTrack` 在内部组合调用 `Ctrl`，即一个算法单元依赖另一个算法单元。这符合《0-HLD.md》5.2 的规则——算法单元之间允许最小化、无循环的依赖。`Ctrl` 是被复用的单轴原子控制律，依赖方向单一（`PosTrack → Ctrl`），不构成循环。

测试用例设计：

- 给定目标与当前状态：偏差、坐标变换、加速度结果可用简单几何核对。
- 前向（长机型 `kp=ki=0`，默认 `kiv=0`）：地速标量误差由 `kd` 响应（纯 P），前向位置偏差不产生前向加速度；置 `kiv` 非零可上速度积分。
- 前向（僚机型 `kiv=0`）：前向加速度同时受待飞距（`kp/ki`）与地速误差（`kd`）驱动。
- 垂向/右侧向：位置偏差 + 速度偏差按 PID 响应，其中 P 响应位置偏差、I 累积位置偏差、D 响应速度偏差。
- 输出限幅生效；`reset` 后积分清零。
- 地速 < `vMin`：`step` 报错中止，不产生加速度。

## 2.3 控制算法 Ctrl

作用：单轴原子控制律。被 `PosTrack` 在内部组合调用，**不直接绑定 `Context`**，输入输出为标量（不受端口绑定粒度限制，因为它不参与 L1 串联）。

### 2.3.1 抽象类

```python
class CtrlBase:
    def init(self, cfg: "CtrlInitS") -> None: ...
    def step(self, posErr: float, velErr: float) -> float: ...  # 返回该轴控制量（加速度分量）
    def reset(self) -> None: ...

@dataclass
class CtrlInitS:
    kp: float = 0.0        # 位置误差比例
    ki: float = 0.0        # 位置误差积分
    kd: float = 0.0        # 速度误差比例（阻尼）
    kiv: float = 0.0       # 速度误差积分；与 ki 互斥，二者不可同时非零
    dt: float = 0.0        # 控制周期，单位 s
    iMax: float = 0.0      # 位置积分限幅（0 表示不限）
    iMaxVel: float = 0.0   # 速度积分限幅（0 表示不限）
    outMax: float = 0.0    # 输出限幅（0 表示不限）
```

`dt` 必须等于本控制律实际被调用的控制周期。播放倍率和 UI 刷新率不改变 `dt`；修改 `step_s` 或 `algorithm_decimation` 会改变 `dt`，并会影响积分项累积速度和闭环动态。

### 2.3.2 子类：PID

实现方法：通用双通道控制律 `输出 = kp*posErr + ki*积分(posErr) + kd*velErr + kiv*积分(velErr)`。位置误差走 `kp/ki` 通道、速度误差走 `kd/kiv` 通道，两路积分相互独立。`velErr` 既作微分项（`kd`，速度误差即位置误差的导数，避免对误差做数值差分），也可单独闭环（`kiv`，用于无位置基准、只控速度的场景）。两路积分各自做积分限幅（`iMax` / `iMaxVel`），最终输出做输出限幅。积分值与上一拍量存于 `self`。

> **互斥保护**：`ki` 与 `kiv` 不可同时非零——位置积分与速度积分同时投入会形成双积分器互相绕死。`init` 在配置阶段检测到二者均非零时直接报错。由此一套控制律可统一覆盖两种角色：长机置 `kp=ki=0`，前向为速度环（速度比例走 `kd`；`kiv` 速度积分通道预留，置非零即上 PI，**当前默认 `kiv=0` 即纯速度 P**）；僚机置 `kiv=0`（位置走 `kp/ki`、`kd` 作阻尼）为位置 PID。

接口：实现 `init/step/reset`。`reset` 清零两路积分。

测试用例设计：

- 纯 P（ki=kd=kiv=0）：输出 = kp*posErr。
- 位置 I：多拍累积，验证位置积分增长与 `iMax` 限幅。
- 速度通道（kd/kiv）：`kd` 响应速度误差，`kiv` 累积速度误差并受 `iMax Vel` 限幅。
- 互斥保护：`ki` 与 `kiv` 同时非零时 `init` 报错。
- 输出限幅：大误差时输出被限到 `outMax`。
- `reset`：两路积分清零。

### 2.3.3 扩展子类

- L1：自适应 L1 控制律。
- ADRC：自抗扰控制律。
