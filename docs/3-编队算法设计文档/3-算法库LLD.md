# 算法库 LLD

> 算法库是一组**不碰 `Mode` 的纯计算类**（实例态在 `self`、无全局态），跨方法复用；由流程在外部按 Mode 选 / 切（分界见 `0-HLD.md` 原则 9）。
> 单元小协议见 `1-LLD综述.md` §2.1：`init(params, bind) / step(self) / reset / read_state`——`step(self)` 经**已绑定的黑板视图**读写（无传值 u/y）。
> 策略族按"**共性父类 / 差异子类、输入下沉 / 输出族级**"组织（`1-LLD综述.md` §2.2）。**每新增一个算法，在此册增改对应单元。**

## 1. 单元清单（4 个策略族 + 通用数学）

每个族给"读黑板槽 → 写黑板槽"。**输入下沉**：长 / 僚机实现各只声明它真读的槽（不为同族拉齐）；**输出族级**：下游必读的输出槽留父类。

| 族（抽象基类） | 单元 | 读黑板 → 写黑板 | 本轮 |
| --- | --- | --- | --- |
| `PositionSolve` | 位置解算 | **父类输出槽**：`目标指令`；**航线插值**(长机)：`航段, 本体运动状态` → `目标指令, 待飞距`；**槽位几何**(僚机)：`队形指令, 主机运动状态`(+init 飞机ID/队形信息) → `目标指令` | ✅ |
| `DeviationCalc` | 误差解算 | `目标指令, 本体运动状态` → `三轴误差`（三轴位置误差 + 速度误差；航线插值时前向位置误差为 0） | ✅ |
| `Tracking` | 编队跟踪 | `三轴误差` → `三轴加速度`(=control)；**按通道组合控制算法** | ✅ |
| `ControlLaw` | 控制算法 | 控制误差 → 控制量（被 `Tracking` 组合内部调，不在顶层挂黑板） | ✅ |
| —（工具，非 `step` 单元） | 通用数学 | 坐标变换、限幅、滤波；（未来）一致性律 | 按需 |

- **位置解算实现**：航线插值（长机，吃 `航段` + `本体运动状态` 做投影 / 待飞距）/ 槽位几何（僚机，吃 `队形指令` + `主机运动状态`（长机参考）+ init 几何）/（未来）Dubins。两实现**输入下沉、不拉齐**，统一在输出槽 `目标指令`（`待飞距` 仅航线插值写，是反馈槽）。**选哪个实现由挂接定，不在单元内 `if 航段空` 分支**（原则 10）。
- **跟踪实现**：不同通道组合（如垂向 TECS、横侧 PID）；跟踪本身不挑模态，挑哪套组合由流程在外部切。
- **控制算法实现**：PID / L1 / ADRC 等原子律。

> 输出口径对齐 `../2-模型迭代HLD.md`：跟踪链最终产出 `AccelerationCommand{ax/ay/az_cmd_mps2}`（ENU），写黑板 `三轴加速度` 槽。

## 2. 单元规范（统一）

- **类、无全局态**：可变状态放 `self`；同一实体可挂同类多个实例（如三轴各一个控制算法），互不干扰。
- **不碰 `Mode`**：不读任务状态机；"用哪个策略"由流程在外部选 / 切（含位置解算两实现按挂接选），单元只管纯计算。
- **黑板视图**：`step(self)` 经 `init` 绑定的 In/Out 指针读写黑板；In 是 impl 私有（下沉），Out 的下游必读槽在父类（族级）。
- **C 友好**：`PID(gains)` ≡ `pid_init(pid_t*, gains)`，`self.pid.step(e)` ≡ `pid_step(pid_t*, e)`。
