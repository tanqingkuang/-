# 算法库 LLD

> 算法库是一组**不感知状态机的纯计算类**（实例态在 `self`、无全局态），跨方法复用。
> 单元小协议见 `1-LLD综述.md` §3：`init(params) / step(inputs)->outputs / reset`。
> **每新增一个算法，在此册增改对应单元。**

## 1. 单元清单

| 类别 | 单元 | 输入 → 输出 | 本轮 |
| --- | --- | --- | --- |
| 控制 | PID（实例持积分 / 限幅态） | 误差 → 控制增量 | ✅ |
| 制导律 | 比例导引 / 视线制导 / 相对位置制导 | 目标 + 本机状态 → 期望加速度 | ✅（至少一种） |
| 编队几何 | 槽位解算 | 槽位号 + 队形类型 + 长机位 → 期望点 | ✅ |
| 航迹偏差 | 误差解算 | 本机状态 + 目标 → 侧偏 / 待飞距 / 航迹角偏差 / 曲率 | ✅ |
| 航线生成原语 | 航线插值；（未来）Dubins / 集结航线 / 避障航线 | 航点序列 → 当前目标 | 部分 |
| 通用数学 | 坐标变换、限幅、滤波；（未来）一致性律 | — | 按需 |

> 输出口径对齐 `../2-模型迭代HLD.md`：跟踪 / 制导链最终产出 `AccelerationCommand{ax/ay/az_cmd_mps2}`（ENU）。

## 2. 单元规范（统一）

- **类、无全局态**：所有可变状态放 `self`；同一实体可挂同类多个实例（如三轴各一个 PID），互不干扰。
- **纯计算、不感知模态**：不读任务状态机；模态分支属流程库 / 编排。
- **C 友好**：`PID(gains)` ≡ `pid_init(pid_t*, gains)`，`self.pid.step(e)` ≡ `pid_step(pid_t*, e)`。
