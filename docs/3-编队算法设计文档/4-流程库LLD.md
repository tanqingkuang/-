# 流程库 LLD

> 流程库是一组**碰 `Mode` 的流程类**（与算法库平级，区别在于它产 / 用 / 切任务模态；分界见 `0-HLD.md` 原则 9）。
> 单元小协议见 `1-LLD综述.md` §2.1。**每新增一个流程实现，在此册增改对应单元。**

## 1. 单元清单

我们原先设想的"6 层"大部分一落地就**塌进两边**：不碰 Mode 的纯计算（位置解算 / 误差解算 / 跟踪 / 控制算法）归算法库，按 Mode 串联接线（任务执行）本轮退化进实体 `step()`。真正属于"流程"且本轮需要的：

| 单元 | 本轮 | `u` → `y` | 说明 |
| --- | --- | --- | --- |
| 收发处理(收 / 发) | ✅ | `inbox` → `ParsedInbox` / 广播内容 → envelope | topic 与 payload 映射 |
| 任务编排 | ✅（占位） | `mission_command \| None` → `Mode` | 本轮恒"保持" |
| 轨迹规划 | ✅ | `(Mode, ParsedInbox \| self_state)` → `Plan` | mode-aware：长机航线推进（航点切换 / 待飞距驱动）、僚机选队形 + 槽位；把 `Plan` 交给算法库位置解算 |
| 任务执行 / 模态分支 | ⏸ 退化 | — | 按 Mode 选策略 / 串联，本轮写死在 `step()`；含金量在此，待模态丰富 |

> **结论**：本轮流程库 = 收发 + 编排(占位) + 轨迹规划，加退化进 `step()` 的任务执行。"流程库"作为会随模态长起来的东西，先留壳。
