# 流程库 LLD

> 流程库是一组**碰 `Mode` 的流程类**（与算法库平级，区别在于它产 / 用 / 切任务模态；分界见 `0-HLD.md` 原则 9）。
> 与算法库同构：**每个流程单元都是一个策略族**（抽象基类 + 可换实现，按"共性父类 / 差异子类、输入下沉 / 输出族级"组织），选哪个实现由实体挂接时决定（`0-HLD.md` 原则 10）。
> 单元小协议见 `1-LLD综述.md` §2.1：`step(self)` 经**已绑定的黑板视图**读写；收 / 发消息的跨实体载荷仍是 `list[MessageEnvelope]`。**每新增一个流程实现，在此册增改对应单元。**

## 1. 单元清单

我们原先设想的"6 层"大部分一落地就**塌进两边**：不碰 Mode 的纯计算（位置解算 / 误差解算 / 跟踪 / 控制算法）归算法库，按 Mode 串联接线本轮退化进实体 `step()`（固定顺序，不单列单元）。真正属于"流程"且本轮需要的：

| 族（抽象基类） | 单元 | 本轮 | 读黑板 → 写黑板（角色） | 同族可换实现 |
| --- | --- | --- | --- | --- |
| `Inbound` | 收消息 | ✅ | 僚机：`inbox(list[MessageEnvelope])` → `主机运动状态, 编队模式, 队形指令`；长机本轮不收 | 领航-跟随解析 / 点对点 / … |
| `Outbound` | 发消息 | ✅ | 长机：`队形指令, 主机运动状态, 编队模式`（+init 网络拓扑）→ `outbox(list[MessageEnvelope])`；僚机不发（空列表） | 领航-跟随广播 / 点对点 / 不发 |
| `Orchestrate` | 任务编排 | ✅（占位） | 长机：`遥控指令(=mission_command), 上一拍编队模式` → `编队模式, 队形指令`；僚机无编排（模态由收消息出） | 常量"保持"（本轮）/ 真实模态状态机 |
| `TrajectoryPlan` | 轨迹规划 | ✅ | `编队模式, 上一拍航段, 本体运动状态, 上一拍待飞距` → `航段`；长机航线推进（航点切换 / 待飞距驱动）、僚机空规划（预埋，本场景产空航段） | 航线推进（长机）/ 空规划（僚机）/ 未来集结·重构规划 |

> **结论**：本轮流程库 = 收消息 + 发消息 + 编排(占位) + 轨迹规划；按模态串联各单元退化进 `step()`（固定顺序，不单列）。"流程库"作为会随模态长起来的东西，先留壳。
> 原"队形规划"塌进 `发消息`：本轮队形 / 槽位是静态 config、无规划动作；**动态队形 / 槽位分配**（集结 / 重构时解分配问题）留编排侧未来单元。
> 僚机 `轨迹规划` 本场景产**空航段**，纯为：① 让 `位置解算` 族输出契约统一（输出族级）；② 预埋集结 / 重构真实规划。它读 `编队模式` 是流程单元、允许碰 Mode。
