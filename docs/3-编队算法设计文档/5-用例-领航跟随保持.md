# 用例：领航-跟随 编队保持

> 一个 **usecase = 一个待验证方案/场景**，说明它如何把三模块（实体组 + 算法库 + 流程库）**组合**成可跑的配置。
> 本册是本轮场景：**集中式任务编排 + 集中式通信 + 长机-僚机 + 仅编队保持**。
> 架构见 `0-HLD.md`，契约见 `1-LLD综述.md`，单元实现见 `3-算法库LLD.md` / `4-流程库LLD.md`。

## 1. 实体与装配

| 实体 | 挂接的单元（执行顺序） | 产出 |
| --- | --- | --- |
| **长机本体** | 飞行：`任务编排 → 轨迹规划(航线推进) → 位置解算(航线插值) → 误差解算 → 跟踪`；协调：`收发(发)`（组装 `leader_nav + 队形 + 槽位` 广播） | `control`（长机三轴加速度）+ `outbox`（广播） |
| **僚机本体 ×N** | `收发 → 任务编排 → 轨迹规划(选队形+槽位) → 位置解算(槽位几何) → 误差解算 → 跟踪` | `control`（僚机三轴加速度） |

长机本体在**一个对象**内同时跑"飞自己"和"协调别人"两条流，共享自己的状态、直接接线，无绕路。长机飞行流与僚机飞行流几乎同构，只差"产出待跟踪目标"那几步的来源：

```
长机：           轨迹规划(航线推进)→位置解算(航线插值)   → 误差解算 → 跟踪 → control
僚机：收发 → 轨迹规划(选队形+槽位)→位置解算(槽位几何)   → 误差解算 → 跟踪 → control
                ↑ 都产出 Target，下游 误差解算 / 跟踪 完全一致
```

**任务编排本轮内联（占位），先不抽离；各单元的串联本轮就是 `step()` 里写死的固定顺序：**

| 动作 | 本轮放哪 | 本轮形态 | 将来触发 |
| --- | --- | --- | --- |
| 任务编排（定模态） | 各实体的编排单元（长机读注入 `mission_command`、僚机读广播 `task`） | 写死返回"保持"的常量 | 出现真实模态决策 |
| 串联（按模态选管线） | 每个实体的 `step()`（固定顺序，不单列单元） | 写死固定调用顺序 | 出现按模态换管线 / 异构僚机 |

> **僚机扩展靠实例化**（`[Wingman(cfg_i) for cfg_i in configs]`），串联写死在 `step()` 是类级共享代码、不写死个数；与编排丰富正交。

## 2. step 装配

```python
# 僚机本体
def step(self, ctx):
    parsed = self.rx.step(ctx.inbox)                       # 收发(收)
    mode   = self.orch.step(parsed)                        # 任务编排(僚机 Mode 来自广播, 恒"保持")
    plan   = self.planner.step((mode, parsed))             # 轨迹规划(选队形+槽位)
    target = self.possolve.step((plan, parsed.leader_nav)) # 位置解算(槽位几何)
    dev    = self.devcalc.step((ctx.self_state, target))   # 误差解算
    accel  = self.tracker.step((dev, ctx.self_state))      # 跟踪
    return FormationAlgorithmOutput(control=accel, outbox=[], status=self._status())

# 长机本体（飞行 + 协调）
def step(self, ctx):
    # 飞行流
    mode   = self.orch.step(ctx.mission_command)           # 任务编排(恒"保持")
    plan   = self.planner.step((mode, ctx.self_state))     # 轨迹规划(航线推进)
    target = self.possolve.step(plan)                      # 位置解算(航线插值)
    dev    = self.devcalc.step((ctx.self_state, target))   # 误差解算
    accel  = self.tracker.step((dev, ctx.self_state))      # 跟踪
    # 协调流（编排已在飞行流产出 mode；队形 / 槽位是 self 的静态 config）
    bcast  = self.tx.step((ctx.self_state, mode))          # 收发(发)：组装 leader_nav + 队形 + 槽位 → envelope
    return FormationAlgorithmOutput(control=accel, outbox=[bcast], status=self._status())
```

## 3. 长机→僚机消息 payload（领航-跟随插件自声明）

comm 只认通用 `MessageEnvelope`，不解析 payload。长机每个协调节拍由 `收发(发)` **广播一条**：

- `topic`: `"lead_broadcast"`；`source`: 长机 ID；`target`: `"*"`（广播）
- `payload`:

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `task` | `str` | 当前任务 / 模态，本轮恒 `"formation_keep"` |
| `leader_nav` | `NavState` | 长机导航量（位置 E/N/U、V、θ、ψ） |
| `formation_type` | `str` | 队形类型，如 `"wedge"` / `"line"` / `"echelon"` |
| `slot_assignment` | `dict[str, int]` | `{僚机 ID: 槽位号}` |

僚机的 `位置解算`（算法库，槽位几何实现）用 `leader_nav + formation_type + 本机槽位号` → 期望目标点。**几何（槽位号 → 相对偏置）是队形定义的一部分，长机 `收发(发)` 与僚机共享同一份队形几何（算法库）**；长机只发"谁是几号槽"、不发具体偏置（粒度=槽位级）。
