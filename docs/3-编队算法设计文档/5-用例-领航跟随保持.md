# 用例：领航-跟随 编队保持

> 一个 **usecase = 一个待验证方案/场景**，说明它如何把三模块（实体组 + 算法库 + 流程库）**组合**成可跑的配置。
> 本册是本轮场景：**集中式任务编排 + 集中式通信 + 长机-僚机 + 仅编队保持**。
> 架构见 `0-HLD.md`，契约见 `1-LLD综述.md`（黑板 4 层 + 各单元 In/Out），单元实现见 `3-算法库LLD.md` / `4-流程库LLD.md`。

## 1. 实体与装配

| 实体 | 挂接的单元（执行顺序） | 产出 |
| --- | --- | --- |
| **长机本体** | 飞行：`任务编排 → 轨迹规划(航线推进) → 位置解算(航线插值) → 误差解算 → 跟踪`；协调：`发消息`（组装 `长机运动状态 + 编队模式 + 队形指令` 广播） | `control`（长机三轴加速度）+ `outbox`（广播） |
| **僚机本体 ×N** | `收消息 → 轨迹规划(空规划预埋) → 位置解算(槽位几何) → 误差解算 → 跟踪`（**无任务编排**：模态由收消息解出） | `control`（僚机三轴加速度） |

长机本体在**一个对象**内同时跑"飞自己"和"协调别人"两条流，共享同一份黑板、直接读写，无绕路。长机飞行流与僚机飞行流几乎同构，只差"产出待跟踪目标"那几步的来源：

```
长机：            轨迹规划(航线推进)→位置解算(航线插值)   → 误差解算 → 跟踪 → control
僚机：收消息  →   轨迹规划(空预埋)  →位置解算(槽位几何)   → 误差解算 → 跟踪 → control
                 ↑ 都产出 目标指令(写同一黑板槽)，下游 误差解算 / 跟踪 完全一致
```

**任务编排本轮内联（占位），先不抽离；各单元的串联本轮就是 `step()` 里写死的固定顺序：**

| 动作 | 本轮放哪 | 本轮形态 | 将来触发 |
| --- | --- | --- | --- |
| 任务编排（定模态） | 长机的编排单元（读注入 `mission_command` + 上一拍编队模式）；**僚机不跑编排**，模态由 `收消息` 从广播解出 | 写死返回"保持"的常量 | 出现真实模态决策 |
| 串联（按模态选管线） | 每个实体的 `step()`（固定顺序，不单列单元） | 写死固定调用顺序 | 出现按模态换管线 / 异构僚机 |

> **僚机扩展靠实例化**（`[Wingman(cfg_i) for cfg_i in configs]`），串联写死在 `step()` 是类级共享代码、不写死个数；与编排丰富正交。

## 2. step 装配（黑板版；u/y 已在 init 绑到黑板）

```python
# 僚机本体（无任务编排 / 发消息）
def step(self, ctx):
    self._ingest(ctx)        # self_state→本体运动状态（inbox 不入黑板）
    self.rx.step(ctx.inbox)  # 收消息(例外签名)：inbox → 主机运动状态, 编队模式, 队形指令
    self.planner.step()      # 轨迹规划(空预埋)：→ 航段(空)
    self.possolve.step()     # 位置解算(槽位几何)：队形指令+主机运动状态 → 目标指令
    self.devcalc.step()      # 误差解算：目标指令+本体运动状态 → 三轴误差
    self.tracker.step()      # 跟踪：三轴误差 → 三轴加速度
    return self._emit()      # control=三轴加速度, outbox=[]

# 长机本体（飞行 + 协调，同一对象 / 同一黑板）
def step(self, ctx):
    self._ingest(ctx)        # time_s/dt_s→黑板时钟；self_state→本体运动状态(==主机运动状态)、mission_command→遥控指令
    # 飞行流
    self.orch.step()         # 任务编排：遥控指令+上一拍编队模式 → 编队模式, 队形指令
    self.planner.step()      # 轨迹规划(航线推进)：→ 航段
    self.possolve.step()     # 位置解算(航线插值)：航段+本体运动状态 → 目标指令, 待飞距
    self.devcalc.step()      # 误差解算：目标指令+本体运动状态 → 三轴误差
    self.tracker.step()      # 跟踪：三轴误差 → 三轴加速度
    # 协调流（共享同一黑板：编排已写 编队模式/队形指令；网络拓扑是 init 静态）
    self.tx.step()           # 发消息：队形指令+主机运动状态+编队模式+time_s → outbox(广播 1 条)
    return self._emit()      # control=三轴加速度, outbox=广播
```

## 3. 长机→僚机消息 payload（领航-跟随插件自声明，粒度 = 飞机 id 级）

comm 只认通用 `MessageEnvelope`，不解析 payload。长机每个协调节拍由 `发消息` **广播一条**：

- `topic`: `"lead_broadcast"`；`source`: 长机 ID；`target`: `"broadcast"`（广播，对齐 `src/runner/sim_control.py` 约定）
- `payload`: 采用与 `4-流程库LLD.md` 一致的 canonical schema，序列化字段名使用 ASCII key：

| 逻辑字段 | payload key | 类型 | 说明 |
| --- | --- | --- | --- |
| `编队模式` | `mode` | `Mode` | 当前任务 / 模态，本轮恒 `保持` |
| `长机运动状态` | `leader_state` | `NavState` | 长机导航量（位置 E/N/U、V、θ、ψ）；僚机收作 `主机运动状态` |
| `队形指令` | `formation_command` | `FormationCommand` | 队形类型，如 `"三角"` / `"wedge"` / `"line"` |

示例：

```json
{
  "mode": {"phase": "保持"},
  "leader_state": {
    "node_id": "A01",
    "x_m": 140.0,
    "y_m": 260.0,
    "altitude_m": 1200.0,
    "speed_mps": 5.2,
    "theta_rad": 0.0,
    "psi_rad": 0.0
  },
  "formation_command": {"shape": "wedge"}
}
```

僚机的 `位置解算`（算法库，槽位几何实现）用 `主机运动状态 + 队形指令 + 本机飞机ID` → 期望目标点。**几何（飞机ID → 相对偏置）是队形定义的一部分，长机 `发消息` 与僚机共享同一份 `队形信息`（算法库，`init` 静态注入）**；长机只发"队形类型"、不发具体偏置（**粒度 = 飞机 id 级**：用静态 `队形信息` 按 `飞机ID` 索引，不发 `slot_assignment`）。
