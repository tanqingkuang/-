# 用例 - 队形切换（多队形 + 运行时热切换）

## 1. 目标

在长机-僚机 + 集中式通信 + PPI/pos_track 制导的既有架构上，支持：

- 一份配置里定义**多种队形**（离线规划）；
- 仿真运行中从**界面选择目标队形**，长机据此**热切换**（不重建模块、不复位时间）；
- 僚机随长机广播当拍切到对应槽位。

演示配置见 `configs/change.json`（5 机、直线航线、25 m/s、初始队形 0）。

## 2. 队形标识：纯整型索引

`cmd.pattern` 是**纯整型队形索引**（0 起），直接作为 `formPos` 的行号。代码不认死任何队形名——
不同配置的队形各不相同，语义完全由配置里的**队形列表顺序**决定：

- 界面显示：队形**名字**（字符串，观察者友好）；
- 算法/通信：只传**整型索引**（长机只知道"切到第 i 个"，不关心它叫什么）。

> 历史的 `FormPatE` 枚举已删除。旧的单队形配置（`formation.pattern` + `formation.slots`）仍兼容，等价于"只有一个队形、初始索引 0"。

## 3. 配置结构（多队形）

```json
"formation": {
  "coordinate_system": "x_forward_y_up_z_right",  // 必填，长机航迹系 x前/y上/z右
  "reference_node_id": "A03",
  "initial_index": 0,                              // 初始队形索引（cmd.pattern 初值）
  "formations": [                                  // 索引 = 列表顺序
    { "name": "楔形",   "slots": [ {"node_id":"A01","x_m":-90,"y_m":0,"z_m":-90}, ... ] },
    { "name": "横队",   "slots": [ ... ] },
    { "name": "双纵队", "slots": [ ... ] }
  ]
}
```

- 每个队形都必须给**全部节点**的槽位，缺一即报错（解析在 `sim_control_routes._build_multi_formations` / `_parse_slot_list`）。
- 解析结果打包进 `FormCommInitS`：`formPat=[各队形名]`、`formPos=[[各队形槽位]×M]`、`initialPattern=initial_index`，init 阶段一次性下发给每架飞机。

## 4. 切换数据流（热切换，不重建）

```
“场景”下拉(SelectButton.currentIndexChanged) → MainWindowActionMixin._on_scenario_selected()
  → ControllerSimulationAdapter.switch_formation(i)
    → SimulationController.switch_formation(i)          # 运行态可用；定位长机 Hold 任务
      → Hold.set_pattern_index(i)                       # 改长机目标队形索引
        → 下一算法拍 Hold.step 写出 cmd.pattern=i
          → LeaderBroadcast 广播 formation.leader(含 pattern=i)
            → 僚机 LeaderFollower 入站解析 cmd.pattern=i
              → SlotGeometry.step 取 formPos[i] 的本机槽位 → 新目标位
```

- 界面复用既有“场景”下拉（`SelectButton`）作为队形选择器，不新增控件；`i` 从 `scenario_select.currentIndex()` 读取。
- 全程**不重建模块、不复位时间**：多队形槽位表 init 时已整份下发，切换只改一个整数。
- 界面队形名列表由 `SimulationController.get_formation_names()`（← `formation_comm_init.formPat`）提供、初始选中项由 `get_formation_index()` 提供，配置加载后 `_refresh_formation_options()` 刷新下拉。
- 长机 `Hold` 不再写死队形，`init` 时置 `initialPattern`，`reset` 回到初始索引。

## 5. 避撞（当前：4.1 离线摆槽位）

本版避撞只靠**离线摆槽位**：各队形间同一架机不换边、不穿越中线（`change.json` 里 A01/A02 恒在左、A04/A05 恒在右，长机 A03 居中），直接切期望位置让 pos_track 收敛。

后续可选项（未落地）：分层高(4.2)、扩散→切→收敛(复用 `ScaledSlotGeometry`，4.3)、机间避撞前馈(4.4)。
