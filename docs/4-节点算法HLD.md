# 节点算法 HLD

## 1. 定位

节点算法属于验证对象组，每架飞机本地运行一个实例，承载编队方法插件、任务流程骨架、制导和控制律，是后续移植到 C 的核心代码。

## 2. 职责

- 接收仿真控制注入的本机状态、邻居消息、任务阶段和配置。
- 执行集结、保持、重构、避障等阶段下的本机算法。
- 输出控制量和待发送消息。
- 声明算法插件的消息 schema。
- 保持无 I/O、不持引擎引用，形成消息驱动任务形态。

## 3. 边界

- 不直接调用模型迭代或通信功能。
- 不直接读取配置文件或写日志。
- 不处理链路时延、丢包、断链等 QoS / 扰动行为。
- 不持有其他节点算法实例引用。

## 4. 主要接口类别

- 生命周期：`init`、`close`
- 算法主接口：`step`
- 消息输入：`deliver_inbox`
- 消息输出：`read_outbox`
- 控制输出：`read_control`
- schema 声明：`declare_message_schema`
