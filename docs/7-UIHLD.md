# UI HLD

## 1. 定位

UI 是人机交互组的 Boundary，包含控制界面和实时显示，面向带界面的单次仿真。UI 负责配置展示、参数编辑、运行控制、实时状态展示和动态扰动注入入口。

## 2. 职责

- 加载并展示基础配置对象。
- 支持用户编辑算法选择、航线、队形、扰动、日志等配置项。
- 调用仿真控制启动、暂停、恢复、单步、停止仿真。
- 将用户运行期动态扰动命令转发给仿真控制。
- 接收仿真控制推送的实时状态快照并展示。
- 为后续二维 / 三维可视化预留扩展点。

## 3. 边界

- 不直接调用协调算法、节点算法、模型迭代、通信功能或加扰。
- 不直接写关键数据日志。
- 不实现批量仿真调度。
- 不在 headless 模式启用。

## 4. 主要接口类别

- 配置操作：加载、展示、编辑、导出配置对象
- 运行控制：`start`、`pause`、`resume`、`step`、`stop`
- 动态扰动：`inject_disturbance`
- 实时显示：状态流订阅、刷新、告警展示
