# 数据分析设计文档

## 1. 目标与边界

数据分析功能用于离线评估仿真输出的控制效果。功能入口只读取 `snapshots.jsonl`，不参与仿真闭环，不回读配置文件推导理论队形，也不解释扰动机理。

本功能把扰动、参数或配置差异产生的日志当作测试结果，重点回答：

- 指定时间段内，所有飞机或某一架飞机的跟踪误差统计结果。
- 六个默认误差通道中，哪个方向的误差水平、波动或最坏值更明显。
- 滑动窗口曲线中，问题大致出现在什么时间段，是否持续或恢复。
- 两份日志在相同对象、时间段、窗口宽度和绘图通道下的并列差异。

明确不做：

- 不读取 `config.json` 或 `events.jsonl`。
- 不修改 `SimulationSnapshot`、`NodeState` 或日志写入格式。
- 不做扰动原因归因、航段识别或自动结论。
- 不做 `B - A`、`B / A`、更优项标记、阈值高亮等进一步数值比较。
- 不导出滑动窗口曲线数据。

## 2. 用户入口与界面结构

正式入口在 PySide6 主窗口菜单中提供：

```text
数据分析 -> 控制效果分析
```

低保真界面示意如下。该图只表达信息结构和主要交互，正式运行界面以 PySide6 实现为准。

![数据分析界面草图](assets/数据分析界面草图.svg)

界面按上中下组织：

| 区域 | 内容 | 设计约束 |
| --- | --- | --- |
| 顶部输入区 | 文件 A/B、启用勾选、分析对象、开始时间、结束时间、窗口宽度 | 文件 A 默认启用，文件 B 默认不启用；修改参数后自动刷新 |
| 中部汇总区 | 六个默认通道的指标表、全部指标导出按钮 | 表格始终展示全部默认通道；绘图通道不影响该表 |
| 底部曲线区 | 左侧绘图通道单选，右侧四张滑动窗口曲线 | 四张图分别展示均值、标准差、RMS、最大绝对值，各自独立 Y 轴 |

曲线标题旁提供弹出按钮。弹出窗口只显示当前四张滑动窗口曲线，沿用当前文件启用状态、分析对象、时间段、窗口宽度和绘图通道。

## 3. 输入输出契约

### 3.1 输入文件

输入只支持 `snapshots.jsonl`。每一行需要包含：

| 字段 | 说明 |
| --- | --- |
| `time_s` | 当前快照仿真时间，单位 s |
| `nodes` | 节点状态列表 |
| `nodes[].node_id` | 飞机 ID |
| 默认误差通道 | 六个跟踪误差字段 |

默认误差通道如下：

| 字段 | 中文名 | 单位 |
| --- | --- | --- |
| `track_pos_err_x_m` | 前向位置误差 x | m |
| `track_pos_err_y_m` | 垂向位置误差 y | m |
| `track_pos_err_z_m` | 侧向位置误差 z | m |
| `track_vel_err_x_mps` | 前向速度误差 x | m/s |
| `track_vel_err_y_mps` | 垂向速度误差 y | m/s |
| `track_vel_err_z_mps` | 侧向速度误差 z | m/s |

航迹系误差字段是控制诊断量，只报告当前控制参数实际使用的误差信号；某轴位置或速度误差未进入控制律时，该轴对应诊断字段输出 0。

字段缺失或类型错误时，不按 0 填充，而是作为日志契约错误提示给用户。

### 3.2 输入参数

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| 文件 A/B | 文件路径 + 启用状态 | 最多两份 `snapshots.jsonl`；未启用文件不参与展示和导出 |
| `analysis_target` | `all` 或某个 `node_id` | `all` 表示合并所有飞机采样点；节点 ID 表示只分析该飞机 |
| `start_s` | 浮点数 | 分析起点，单位 s |
| `end_s` | 浮点数 | 分析终点，单位 s；若小于起点，内核会归一化时间范围 |
| `window_s` | 浮点数 | 滑动窗口宽度，单位 s |
| `plot_channel` | 默认通道之一 | 只影响滑动窗口曲线，不影响汇总指标表和导出 |

### 3.3 输出内容

| 输出 | 内容 | 说明 |
| --- | --- | --- |
| 汇总指标表 | 当前分析对象在六个默认通道上的统计量 | 每个数值单元采用 `A | B` 结构；对应文件未启用时该侧留空 |
| 滑动窗口曲线 | 当前绘图通道的窗口均值、标准差、RMS、最大绝对值 | A/B 同图分层显示，四个统计量分别使用独立子图 |
| 全部指标 CSV | 已启用文件的全机合并统计和逐飞机统计 | 导出覆盖全部默认通道，不受当前分析对象和绘图通道限制 |

CSV 字段如下：

| 字段 | 说明 |
| --- | --- |
| `input_label` | 文件标签，如 `A`、`B` |
| `source_path` | 源 `snapshots.jsonl` 路径 |
| `scope` | `all` 或 `node` |
| `node_id` | `scope=node` 时为飞机 ID；`scope=all` 时为 `all` |
| `channel` | 原始日志字段名 |
| `channel_label` | 中文展示名 |
| `count` | 有效采样点数量 |
| `mean` | 均值 |
| `variance` | 方差 |
| `std` | 标准差 |
| `rms` | RMS |
| `max_abs` | 最大绝对值 |
| `max_abs_time_s` | 最大绝对值发生时刻 |

## 4. 分析内核设计

分析内核位于 `src.data.control_effect_analysis`，不依赖 PySide6。GUI 只负责采集输入、展示结果和触发导出。

核心数据结构：

| 结构 | 职责 |
| --- | --- |
| `AnalysisChannel` | 描述误差通道的内部 key、中文名、单位和日志字段名 |
| `AnalysisSourceData` | 保存某一份日志解析后的时间范围、节点 ID 和各节点通道采样点 |
| `MetricSummary` | 保存一次统计结果，包括样本数、均值、方差、标准差、RMS 和最大绝对值 |
| `AnalysisResult` | 保存某份日志的全机合并统计和逐飞机统计 |

主要流程：

```text
load_snapshot_samples
  -> 解析 snapshots.jsonl
  -> 校验 time_s、nodes、node_id 和默认通道
  -> 生成 AnalysisSourceData

points_for
  -> 按 analysis_target、channel、start_s、end_s 取样本点

summary_for / calc_summary
  -> 对样本点计算汇总指标

sliding_window
  -> 对当前绘图通道生成窗口统计序列

analyze_source / metric_rows_for_source
  -> 生成全机合并统计和逐飞机统计

write_metrics_csv
  -> 写出全部指标 CSV
```

## 5. 指标口径

### 5.1 汇总指标

对每个通道的有效采样点计算：

| 指标 | 口径 |
| --- | --- |
| `count` | 有效采样点数量 |
| `mean` | 算术均值 |
| `variance` | 总体方差，除以 `N` |
| `std` | 总体标准差，等于 `sqrt(variance)` |
| `rms` | 均方根，等于 `sqrt(mean(x^2))` |
| `max_abs` | 最大绝对值 |
| `max_abs_time_s` | 最大绝对值首次出现的时间 |

标准差用于观察误差围绕均值的波动大小，单位与误差本身一致。RMS 用于观察综合误差水平，同时包含稳定偏差和波动。方差是标准差的平方，单位会变成原单位的平方，因此保留在表格和导出中，但不作为曲线主读数。

### 5.2 分析对象

`analysis_target=all` 时，把所有飞机在同一通道上的有效采样点合并后统计。例如 `track_pos_err_z_m` 的全机指标等价于把所有飞机的侧向位置误差采样点放入同一个样本集合。

`analysis_target=<node_id>` 时，只统计该飞机在同一通道上的有效采样点。

无论选择 `all` 还是某一架飞机，汇总指标表都显示六个默认通道。`plot_channel` 只决定下方滑动窗口曲线当前显示哪个通道。

## 6. 滑动窗口设计

滑动窗口以仿真时间为基准，窗口区间为：

```text
[window_start_s, window_start_s + window_s)
```

窗口锚点来自日志中的实际采样时间，且必须满足：

```text
window_start_s + window_s <= end_s
```

因此尾部不足一个完整窗口的数据不会生成半窗口统计。窗口内没有有效点时跳过该窗口。

计算实现采用滚动统计：

- 输入样本先按时间排序。
- 通过左右指针维护当前窗口内样本范围。
- 通过累计和与平方和计算均值、方差、标准差和 RMS。
- 通过单调队列维护窗口最大绝对值。

该算法避免对每个窗口重新扫描全部样本，复杂度为 `O(n + k)`，其中 `n` 是样本点数量，`k` 是窗口锚点数量。

曲线展示采用四个独立子图：

| 子图 | 说明 |
| --- | --- |
| 窗口均值 | 有符号量，可能为正或负 |
| 窗口标准差 | 非负量，观察波动 |
| 窗口 RMS | 非负量，观察综合误差 |
| 窗口最大绝对值 | 非负量，观察最坏窗口 |

不同统计量不共用同一个 Y 轴，避免最大绝对值量级过大时压扁均值或标准差曲线。

## 7. GUI 刷新与缓存设计

GUI 为文件 A/B 分别维护输入状态和分析结果。文件启用状态直接决定该文件是否参与汇总表、曲线和导出。

刷新规则：

- 选择文件后立即加载并刷新。
- 勾选或取消文件后立即刷新可见结果。
- 切换分析对象后刷新汇总表和曲线。
- 切换绘图通道只刷新曲线，不改变汇总表。
- 修改窗口宽度后刷新曲线。
- 开始时间和结束时间在回车或失焦且校验通过后刷新，避免半输入状态触发分析。

曲线按图层管理。A 图层和 B 图层分别缓存，缓存键包含：

```text
source_id, analysis_target, plot_channel, start_s, end_s, window_s
```

启用 B 时只需要加载或显示 B 图层；关闭 B 时隐藏 B 图层，不重新计算 A。弹出曲线窗口复用同一组分析状态，只改变展示容器。

## 8. 导出设计

导出入口为 `导出全部指标`。导出不受当前 `analysis_target` 和 `plot_channel` 限制，始终覆盖每份已启用文件的：

- 全机合并统计：`scope=all`，每个默认通道一行。
- 逐飞机统计：`scope=node`，每架飞机、每个默认通道一行。

导出使用 UTF-8 with BOM，便于 Excel 直接打开中文表头和中文通道名。

导出中 `channel` 固定写原始日志字段名，保证脚本可稳定处理；`channel_label` 写中文展示名，便于人工阅读。

## 9. 错误处理

错误处理原则是尽早暴露日志契约问题，不静默修正数据：

| 场景 | 处理 |
| --- | --- |
| 文件不存在或无法读取 | 当前文件显示错误，未启用另一文件不受影响 |
| JSON 行损坏 | 抛出带行号的错误 |
| 缺少 `time_s`、`nodes`、`node_id` 或默认通道 | 抛出字段缺失错误 |
| 字段类型不能转换为数值 | 抛出字段类型错误 |
| 指定时间段无样本 | 表格对应指标留空或导出 `count=0` |
| A/B 节点集合不同 | 分析对象下拉使用已加载文件的节点集合并集；某文件缺少所选节点时该侧留空 |

## 10. 测试与验证

分析内核测试覆盖：

- `snapshots.jsonl` 解析。
- 六个默认通道读取。
- `all` 合并统计和单机统计。
- 均值、方差、标准差、RMS、最大绝对值口径。
- 滑动窗口不生成尾部半窗口。
- 滑动窗口滚动算法与朴素算法结果一致。
- CSV 导出字段契约，尤其是 `channel` 和 `channel_label`。

GUI 测试覆盖：

- 数据分析窗口 offscreen 构造。
- 文件 A/B 启用状态对汇总表、曲线和导出的影响。
- 绘图通道切换只影响曲线。
- 汇总表固定展示六个默认通道。
