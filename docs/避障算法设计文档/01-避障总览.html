01 避障总览

本章先回答“工程里的避障到底做了什么”。它不是实时局部避障，而是在二维水平面上离线生成一条新的长机航线，预览满意后再采用。

一句话版

避障流水线是：读取配置里的原始长机航线和障碍物，用户勾选本次启用的障碍，点击生成航线后调用 plan_avoidance_route()，成功则产生预览航线，点击采用后控制器用这条航线替换长机原航线。

只规划长机。僚机会继续按编队算法跟随长机，因此避障输出的本质是“换一条长机路线”。

总体流程图

工程入口

UI 侧有两个解析函数：

parse_avoidance_config(path)
parse_avoidance_params(path)

前者偏显示：读障碍物和膨胀间距。后者偏规划：读长机航点、R、L、分辨率、转向惩罚等参数。

生成航线的关键调用发生在 MainWindow._generate_route()：

result = plan_avoidance_route(
    params.waypoints,
    enabled_obstacles,
    turn_radius_m=...,
    clearance_m=...,
    resolution_m=...,
)

成功后只是预览；真正替换长机航线要再走 apply_avoidance_route()。

关键边界

边界含义
二维水平避障障碍被视为无限高柱体，只改 east/north，不改变高度剖面。
离线生成点击生成航线才规划，不是仿真运行时每帧规划。
A* 不管动力学A* 只找通道，圆弧与可飞性由后续模块兜底。
失败不崩溃失败返回 ERR_AVOID_*，原始航线保持可用。
