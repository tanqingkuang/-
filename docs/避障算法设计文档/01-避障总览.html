01 避障总览

本章先回答“工程里的避障到底做了什么”。它不是实时局部避障，而是在二维水平面上离线生成一条新的长机航线，预览满意后再采用。

一句话版

避障流水线是：展开 route_file 与 obstacles_file，把外部经纬高统一转换为内部 ENU；用户勾选本次启用的障碍后，应用层调用 plan_avoidance_route()。成功结果先用于预览，也可以输出为航线文件；只有点击采用后，控制器才会替换长机原航线。

只规划长机。僚机会继续按编队算法跟随长机，因此避障输出的本质是“换一条长机路线”。

总体流程图

工程入口

数据层和应用层负责把外部文件转换成 GUI 可用的稳定快照：

resolve_config_references(config, path)
load_gui_config(path)

前者展开航线、障碍引用并完成 LLA→ENU；后者输出障碍显示数据、地理原点和规划参数，不把算法内部类型暴露给 GUI。

生成航线的交互入口位于 main_window_avoidance.py：

result = self.sim.plan_avoidance_route(
    params.waypoints,
    enabled_obstacles,
    turn_radius_m=...,
    clearance_m=...,
    resolution_m=...,
)

适配器把调用交给 runner 应用层，再由应用层转换为算法对象。成功后只是预览；真正替换长机航线仍要再走 apply_avoidance_route()。

用户操作闭环

  1. 加载主配置，自动展开航线与障碍文件。
  2. 在避障窗口勾选障碍并调整 R、L、安全距离、栅格和转向惩罚。
  3. 点击“生成航线”，只生成内存预览，不改变控制器。
  4. 预览满意后可“航线输出”，或点击“采用航线”替换长机任务航线。
  5. 点击“重置”清除覆盖，恢复配置中的默认航线。

关键边界

边界含义
二维水平避障障碍被视为无限高柱体，只改 east/north，不改变高度剖面。
离线生成点击生成航线才规划，不是仿真运行时每帧规划。
A* 不管动力学A* 只找通道，圆弧与可飞性由后续模块兜底。
外部 LLA、内部 ENU航线文件面向客户使用经纬高，算法和 GUI 规划画布仍统一使用米制 ENU。
性能不牺牲安全候选预筛后会按全量障碍复核；发现漏筛时自动回退为全量 A* 重规划。
失败不崩溃失败返回 ERR_AVOID_*，原始航线保持可用。
