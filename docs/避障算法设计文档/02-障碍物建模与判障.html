02 障碍物建模与判障

避障的第一个核心不是 A*，而是“什么算被障碍挡住”。项目把这个判断集中在 `obstacle.py`，A* 和可飞性校验都复用它。

数据结构

@dataclass
class ObstacleS:
    id: str
    kind: str  # "circle" | "rect"
    center: PosInEarthS
    radius: float
    min_e: float
    min_n: float
    max_e: float
    max_n: float

圆形障碍用圆心和半径；矩形障碍用 east/north 方向上的最小最大边界。高度字段不参与避障，因为本期只做水平面避障。

膨胀判障

inside() 的语义是：点是否落在“障碍 + clearance”里面，边界算内。

if obstacle.kind == "circle":
    return hypot(east - center.east, north - center.north) \
        <= obstacle.radius + clearance

return (
    min_e - clearance <= east <= max_e + clearance
    and min_n - clearance <= north <= max_n + clearance
)
A* 实际绕开的不是黑色真实障碍，而是橙色虚线代表的膨胀障碍。这样路径即使贴着膨胀边界走，也和真实障碍留出安全距离。

为什么要有 blocked()

def blocked(obstacles, east, north, clearance=0.0):
    return any(inside(obstacle, east, north, clearance)
               for obstacle in obstacles)

A* 要判断格心是否可通行，可飞性要判断圆弧采样点是否触障。两者如果各写一套几何判断，很容易出现“A* 认为安全、圆弧校验认为不安全”的不一致。所以工程把形状判断收敛到同一个函数。

配置到对象
