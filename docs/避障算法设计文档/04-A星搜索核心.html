04 A* 搜索核心

本章只看无转向惩罚的普通分支。这是最标准、最容易理解的一条路径：open 里取 f 最小的格点，扩展邻居，更新 g 和 came_from。

主循环骨架

open_heap = [(heuristic(*start_cell), counter, start_cell)]
g_score = {start_cell: 0.0}
came_from = {}
closed = set()

while open_heap:
    _, _, current = heapq.heappop(open_heap)
    if current == goal_cell:
        return _reconstruct(came_from, current, start, goal, cell_center)
    if current in closed:
        continue
    closed.add(current)

    for di, dj, step in _NEIGHBORS:
        tentative = g_score[current] + step * resolution_m
        if tentative < g_score.get((ni, nj), float("inf")):
            came_from[(ni, nj)] = current
            g_score[(ni, nj)] = tentative
            heappush(open_heap, (tentative + heuristic(ni, nj), counter, (ni, nj)))

变量关系

名词在源码里的形态含义
gg_score[cell]起点到当前格点的已知最短距离。
hheuristic(i, j)当前格心到目标格心的欧氏距离估计。
ftentative + heuristic(...)A* 排序依据，越小越优先扩展。
openopen_heap已经发现、等待扩展的候选。
closedclosed已经出堆并扩展过的格点。
父节点came_from[next] = current用于最终从终点倒推路径。

为什么能找到较短路

Dijkstra 只看已经走了多远，也就是 g。A* 同时看 g 和估计剩余距离 h，所以它会更偏向目标方向。项目使用欧氏距离作为 h，和 8 邻域步长一致，通常不会高估剩余最短路。

当 h=0 时，A* 退化成 Dijkstra；当 h 过大时，搜索会更贪心，但可能牺牲最优性。
