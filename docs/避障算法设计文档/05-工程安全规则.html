05 工程安全规则

真实项目里的 A* 不只是教材公式。这里有几条安全规则，用来避免“算法看似有路，工程上却不可信”的情况。

规则一：端点用精确坐标先判障

if blocked(obstacles, start[0], start[1], clearance_m) \
        or blocked(obstacles, goal[0], goal[1], clearance_m):
    return None

to_cell() 会把真实端点吸附到最近格心。如果端点本身在障碍里，但最近格心碰巧在障碍外，只判格心就会漏掉非法输入。源码先判真实端点，再判吸附后的格心。

规则二：禁止对角切角

if di != 0 and dj != 0 and (
    is_blocked(ci + di, cj)
    or is_blocked(ci, cj + dj)
):
    continue

对角移动时，如果横向邻格或纵向邻格有障碍，就不能从角上斜穿过去。否则路径会像从障碍夹缝里切过去，视觉上可能不穿格心，但物理上不安全。

性能快路径与安全慢路径

优化后的规划器先走便宜路径，只有必要时才扩大计算量；但所有快路径都必须汇入全量复核。这里的目标不是用速度交换安全，而是让常见大范围场景少做与结果无关的工作。

规则三：直线清空就跳过 A*

direct_candidates = _leg_candidate_obstacles(
    start, goal, obstacles, clearance_m, 0.0)

if line_of_sight_clear(start, goal, direct_candidates,
                       clearance=clearance_m):
    raw = [start, goal]
else:
    raw = plan_path(...)

原始航段在膨胀障碍外全程可达时，端点直线已经是该腿的最短路径。直接保留两端既不改变结果，也避免创建大范围网格和运行 open/closed 主循环。

规则四：航段与线段两级候选预筛

层级包围盒用途
航段级起终点盒 + clearance + margin只把可能影响本腿搜索范围的障碍交给 A*。
线段级当前视线段盒 + clearance密集采样时只调用附近候选障碍的 inside()。

两级预筛都只剔除包围盒明确不相交的障碍。polygon 的 bounds 在构造时已缓存，圆和矩形也能常数时间给出包围盒。

规则五：全量复核失败就完整重规划

simplified, causes = simplify_path_with_causes(
    raw, leg_obstacles, clearance=simplify_clearance_m)

if candidate_set_is_smaller and not path_clear_against_all_obstacles(
        simplified, obstacles, simplify_clearance_m, sample_step):
    raw = plan_path(start, goal, obstacles, ...)
    simplified, causes = simplify_path_with_causes(
        raw, obstacles, clearance=simplify_clearance_m)

局部候选可能漏掉“原本不在直线盒内、但绕行后进入新路径附近”的障碍。因此候选路径生成后，必须用全量障碍和去冗余相同的 simplify_clearance_m 复核。只要发现漏筛，就放弃快路径结果，用完整障碍集重新运行 A*。

最终 check_feasibility() 还会按全部真实障碍检查每条直线腿，并可返回 ERR_AVOID_STRAIGHT_HITS_OBSTACLE。它是末端诊断护栏，不替代前面的全量重规划。

规则六：缓存格点判障结果

blocked_cache = {}

def is_blocked(i, j):
    cached = blocked_cache.get((i, j))
    if cached is None:
        east, north = cell_center(i, j)
        cached = blocked(obstacles, east, north, clearance_m)
        blocked_cache[(i, j)] = cached
    return cached

搜索过程中同一个格点可能被不同方向多次检查。缓存只在一次规划内有效，避免跨配置复用旧障碍结果。

规则七：无解和配置错误分开处理

情况行为原因
起点终点在障碍里、通道封死return None这是规划无解，由上层转成 ERR_AVOID_*。
分辨率非正、bounds 非法、网格过大raise ValueError这是输入配置错误，需要明确暴露。
