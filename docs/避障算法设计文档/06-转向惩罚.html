06 转向惩罚

普通 A* 会偏向最短距离，但栅格路径可能频繁锯齿。项目提供转向惩罚，让路径更愿意保持方向，减少后续去冗余和圆弧阶段的压力。

代价公式

def _heading_transition_penalty(previous_dir, current_dir,
                                turn_switch_penalty_m,
                                turn_angle_weight_m):
    if previous_dir is None or previous_dir == current_dir:
        return 0.0
    return turn_switch_penalty_m \
        + turn_angle_weight_m * (delta_deg / 45.0)

turn_switch_penalty_m 是只要换方向就加的固定成本。turn_angle_weight_m 是按转角大小加的成本，45° 为一个单位。

为什么状态要变复杂

无转向惩罚时，状态就是格点：

cell = (i, j)

启用转向惩罚后，同一个格点从东边进入和从南边进入，下一步继续前进的代价不一样，所以状态必须包含进入方向：

State = tuple[int, int, int | None]
state = (i, j, previous_dir)

搜索分支对照

条件open 元素g_score key适合场景
两个惩罚都为 0(f, tie, cell)(i, j)追求传统最短格点路。
任一惩罚大于 0(f, tie, state)(i, j, previous_dir)希望路径更少换向、更平顺。

影响与取舍

优点
减少方向抖动，路径更像飞机愿意飞的骨架。
代价
状态数量变多，同一个格点可能按不同入射方向重复存在。
边界
它只是等效米代价，不替代真正的转弯半径校验。
