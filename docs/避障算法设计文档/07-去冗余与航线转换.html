07 去冗余与航线转换

A* 输出的是格点折线，通常又密又锯齿。工程用候选障碍视线检查删除不必要的中间点，同时记录“哪个障碍逼出了这个拐点”，再把结果交给贴障弧和航线转换。

为什么不能直接用 A* 原始路径

原始格点路径适合解释搜索过程，但不适合直接给长机飞：点太多、方向变化频繁、圆弧转换会变复杂。去冗余的目标是保留必要拐点，把能拉直的锯齿段拉成一条直线。

视线可达检查

def _first_blocker(start, end, obstacles, clearance=0.0):
    candidates = _segment_candidate_obstacles(
        start, end, obstacles, clearance)
    if not candidates:
        return None

    length = hypot(end[0] - start[0], end[1] - start[1])
    steps = max(1, ceil(length / sample_step))
    for k in range(steps + 1):
        t = k / steps
        east = start[0] + (end[0] - start[0]) * t
        north = start[1] + (end[1] - start[1]) * t
        for obstacle in candidates:
            if inside(obstacle, east, north, clearance):
                return obstacle
    return None

def line_of_sight_clear(...):
    return _first_blocker(...) is None

线段先按扩展包围盒筛选候选障碍，密集采样只检查附近对象。这里仍使用 ceil，保证实际采样间距不超过 sample_step。与旧布尔接口不同，内部函数返回首个挡路线障碍，为后续 cause 标签提供来源。

贪心去冗余

result = [points[0]]
causes = [None]
anchor = 0
for i in range(1, len(points) - 1):
    blocker = _first_blocker(
        points[anchor], points[i + 1], obstacles)
    if blocker is not None:
        result.append(points[i])
        causes.append(blocker)
        anchor = i
result.append(points[-1])

从锚点开始尽量往远处看。如果锚点看不到下一个点，就说明当前点必须保留；否则当前点可以跳过。

去冗余不是“删掉所有共线点”这么简单。即使不共线，只要从锚点到远点的直线不穿障碍，也可以删掉中间点。

cause 标签为什么重要

simplify_path_with_causes() 返回与简化点等长的 cause 列表。首末点通常为 None；中间点被保留时，cause 指向阻止锚点继续拉直的障碍。

cause 模式几何含义后续处理
None首末点或没有明确逼停来源。作为普通直线骨架点。
单个圆障碍只有一个拐点由该圆逼出。保留固定 R 倒角，不强行合弧。
连续同一圆、同一绕行方向多个拐点正在沿同一膨胀圆绕行。可折叠为一段贴障大弧。

这种“在判障时记账”的做法比事后根据点到圆心的浮点距离反推来源稳定，也不需要额外容差猜测。

转换为 WayPointInputS

函数职责重要限制
points_to_route()把二维点摆成航点，写入速度和高度。不决策 r，所有 r=0。
bake_obstacle_hug_arcs()把连续贴同一圆的拐点串折叠成膨胀半径大弧。只处理满足条件的圆；失败时安全退化。
check_route_feasibility()校验折叠后真正要飞的直线骨架与显式弧。校验必须发生在补 r 与最终烘焙之前。
assign_transition_radius()给剩余直线-直线内部拐点补 r=turn_radius_m。首末点不补；邻段已有曲线则不补。
bake_transition_arcs()当 allow_arc=True 时把剩余交接半径烘焙成曲率航段。半径过大导致切点越出腿长时保留原折线。

simplify_clearance_m 的意义

A* 搜索使用 clearance_m，去冗余使用 simplify_clearance_m。两者默认相同，但允许分开：搜索时可以保守一点，去冗余时可以按另一套安全边界判断拉直线段。

simplify_clearance_m 还有两项合同：它是候选路径全量复核时使用的安全距离，也是圆形障碍贴障弧的半径增量，即 obstacle.radius + simplify_clearance_m。

调小该值通常能减少拐点，但也会同时改变全量漏筛复核边界和贴障弧半径。它不是单纯的显示或去冗余参数。
