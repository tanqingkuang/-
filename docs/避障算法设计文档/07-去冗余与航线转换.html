07 去冗余与航线转换

A* 输出的是格点折线，通常又密又锯齿。工程接着用视线可达检查删除不必要的中间点，再把拐点变成 `WayPointInputS`。

为什么不能直接用 A* 原始路径

原始格点路径适合解释搜索过程，但不适合直接给长机飞：点太多、方向变化频繁、圆弧转换会变复杂。去冗余的目标是保留必要拐点，把能拉直的锯齿段拉成一条直线。

视线可达检查

def line_of_sight_clear(start, end, obstacles, clearance=0.0):
    length = hypot(end[0] - start[0], end[1] - start[1])
    steps = max(1, ceil(length / sample_step))
    for k in range(steps + 1):
        t = k / steps
        east = start[0] + (end[0] - start[0]) * t
        north = start[1] + (end[1] - start[1]) * t
        if blocked(obstacles, east, north, clearance):
            return False
    return True

注意这里用 ceil，保证采样间距不超过 sample_step。如果用向下取整，短线段中间的细障碍可能被漏掉。

贪心去冗余

result = [points[0]]
anchor = 0
for i in range(1, len(points) - 1):
    if not line_of_sight_clear(points[anchor], points[i + 1], obstacles):
        result.append(points[i])
        anchor = i
result.append(points[-1])

从锚点开始尽量往远处看。如果锚点看不到下一个点，就说明当前点必须保留；否则当前点可以跳过。

去冗余不是“删掉所有共线点”这么简单。即使不共线，只要从锚点到远点的直线不穿障碍，也可以删掉中间点。

转换为 WayPointInputS

函数职责重要限制
points_to_route()把二维点摆成航点，写入速度和高度。不决策 r，所有 r=0。
assign_transition_radius()给直线-直线内部拐点补 r=turn_radius_m。首末点不补；邻段已有曲线则不补。
bake_transition_arcs()当 allow_arc=True 时把交接半径烘焙成曲率航段。半径过大导致切点越出腿长时保留原折线。

simplify_clearance_m 的意义

A* 搜索用 clearance_m，去冗余可以用 simplify_clearance_m。两者默认相同，但允许分开：搜索时可以保守一点，去冗余时可以按另一套安全边界判断拉直线段。

如果去冗余的 clearance 太小，折线可能被拉得更直，但也更接近真实障碍；如果太大，路径会保留更多拐点。
