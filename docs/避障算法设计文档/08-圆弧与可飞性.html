08 圆弧与可飞性

A* 和去冗余只能说明“几何通道存在”。飞机能不能按转弯半径飞出来，还要检查急拐点、腿长和圆弧触障。

三类失败

原因码检查对象含义
ERR_AVOID_TURN_TOO_SHARP拐点偏转角接近掉头，圆弧几何不稳定或不可接受。
ERR_AVOID_LEG_TOO_SHORT相邻拐点之间的腿长一条腿容不下两端圆弧切线占用和直线余度 L。
ERR_AVOID_ARC_HITS_OBSTACLE圆弧采样点直线骨架安全，但圆弧外凸后碰到障碍。

腿长约束

每个内部拐点按转弯半径 R 计算切点到拐点距离：

d = R * tan(abs(deflection) / 2)

每条腿必须装下前后两个拐点的切线占用，再加直线余度 L：

leg_len >= tangent_d[i] + tangent_d[i + 1] + leg_margin_m

圆弧触障为什么必要

A* 和去冗余检查的是折线骨架。飞机实际转弯时，拐点附近不是尖角，而是外切圆弧。这个圆弧可能向障碍一侧外凸，所以可飞性模块会重建圆弧并采样检查。

arc = corner_arc(p_prev, p_corner, p_next, turn_radius_m)
for east, north in _arc_sample_points(arc, turn_radius_m, sample_step):
    obstacle_id = _hit_obstacle(obstacles, east, north, arc_clearance)
    if obstacle_id is not None:
        return FeasibilityResult(ok=False, code=ERR_ARC_HITS_OBSTACLE)

planner.py 如何收口

raw = plan_path(...)
simplified = simplify_path(raw, obstacles, clearance=simplify_clearance_m)
feasibility = check_feasibility(full_xy, obstacles, turn_radius_m=...)
route = points_to_route(full_xy, speed_mps=speed_mps, altitudes=full_alt)
assign_transition_radius(route, turn_radius_m)
if allow_arc:
    route = bake_transition_arcs(route)
return PlanResult(ok=True, route=route, simplified_points=full_xy)

PlanResult 是上层 UI 最关心的边界：成功时带 route；失败时带原因码、详情、腿序号或障碍 id。

高度怎么处理

二维避障只改变水平路径。planner.py 会按水平里程在原始航点高度之间线性插值，保持原航线高度剖面，而不是为了避障新增爬升或下降。
