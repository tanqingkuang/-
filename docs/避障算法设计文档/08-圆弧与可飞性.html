08 圆弧与可飞性

A* 和去冗余只能说明“几何通道存在”。当前实现先把满足条件的圆形绕障折线折叠为贴障大弧，再对真正要飞的直线骨架、固定 R 转弯和显式弧段做后置可飞性校验。

四类不可飞失败

原因码检查对象含义
ERR_AVOID_TURN_TOO_SHARP拐点偏转角接近掉头，圆弧几何不稳定或不可接受。
ERR_AVOID_LEG_TOO_SHORT相邻拐点之间的腿长一条腿容不下两端圆弧切线占用和直线余度 L。
ERR_AVOID_STRAIGHT_HITS_OBSTACLE最终直线腿采样点全量障碍末端复核仍发现直线触障。
ERR_AVOID_ARC_HITS_OBSTACLE圆弧采样点直线骨架安全，但圆弧外凸后碰到障碍。

腿长约束

每个内部拐点按转弯半径 R 计算切点到拐点距离：

d = R * tan(abs(deflection) / 2)

每条腿必须装下前后两个拐点的切线占用，再加直线余度 L：

leg_len >= tangent_d[i] + tangent_d[i + 1] + leg_margin_m

两类圆弧

圆弧半径与表示适用位置
固定 R 交接弧半径为飞机最小转弯半径 R；由直线拐点的 r 或烘焙航段表达。剩余直线—直线拐点。
障碍贴行大弧半径为 obstacle.radius + simplify_clearance_m；用 turnSign 与 center 显式表达。连续绕同一圆形障碍的一串拐点。

A* 和去冗余检查的是折线骨架，飞机实际转弯时圆弧可能向障碍一侧外凸。因此固定 R 圆弧和贴障大弧都要按弧长密集采样，并复用统一 inside() 判障。

arc = corner_arc(p_prev, p_corner, p_next, turn_radius_m)
for east, north in _arc_sample_points(arc, turn_radius_m, sample_step):
    obstacle_id = _hit_obstacle(obstacles, east, north, arc_clearance)
    if obstacle_id is not None:
        return FeasibilityResult(ok=False, code=ERR_ARC_HITS_OBSTACLE)

贴障大弧如何生成

  1. 去冗余阶段根据 cause 标签找到“连续由同一个圆形障碍逼出”的拐点串。
  2. 要求拐点沿圆心极角单调同向、至少两个，并且膨胀半径不小于飞机最小转弯半径 R。
  3. 自由点与圆之间使用点—圆切线；相邻两段贴障弧优先使用圆—圆公切线无拐点衔接。
  4. 检查新弧扫掠角不应明显超过原拐点角度跨度，避免切点选反后沿圆长边绕行。
  5. 贴障弧和两侧衔接直线按真实障碍复核，全部通过才替换原拐点串。
贴障弧当前只处理圆形障碍。polygon 和轴对齐矩形继续使用固定 R 倒角。公切线整体方案不可飞时，会退化为各圆独立、逐段尝试；单段仍失败则保留原拐点，不强行输出危险圆弧。

为什么可飞性必须后置

早期流程先按每个折线拐点的固定 R 检查腿长，可能把“固定 R 碎弧不可行、但沿更大膨胀圆连续飞行可行”的路径提前拒绝。当前流程先折叠贴障弧，再校验真正要输出的航线。

straight_subroutes, explicit_arcs = split_route_segments(route)

for points in straight_subroutes:
    check_feasibility(points, turn_radius_m=R, ...)

for arc in explicit_arcs:
    sample_and_check_arc(arc, obstacles, ...)

check_route_feasibility() 在显式弧处分割航线：剩余直线骨架继续检查直线触障、急转角、腿长与固定 R 圆弧；显式贴障弧按自己的实际半径单独检查。被大弧替换掉的中间拐点不再错误地参与固定 R 腿长约束。

planner.py 如何收口

raw = direct_line_or_candidate_astar(...)
simplified, causes = simplify_path_with_causes(
    raw, leg_obstacles, clearance=simplify_clearance_m)

if not clear_against_all_obstacles(simplified):
    raw = plan_path(start, goal, all_obstacles, ...)
    simplified, causes = simplify_path_with_causes(raw, all_obstacles, ...)

route = points_to_route(full_xy, speed_mps=speed_mps,
                        altitudes=full_alt)
if allow_arc:
    route = bake_obstacle_hug_arcs(route, full_causes, obstacles, ...)

feasibility = check_route_feasibility(route, obstacles,
                                      turn_radius_m=turn_radius_m, ...)
if not feasibility.ok:
    return PlanResult(ok=False, code=feasibility.code, ...)

assign_transition_radius(route, turn_radius_m)
if allow_arc:
    route = bake_transition_arcs(route)
return PlanResult(ok=True, route=route, simplified_points=full_xy)

PlanResult 是上层 UI 最关心的边界：成功时带真正可采用和可输出的 route；失败时带原因码、详情、腿序号或障碍 id。注意顺序是“贴障弧 → 后置校验 → 补剩余 R → 烘焙剩余转弯”。

高度怎么处理

二维避障只改变水平路径。planner.py 会按水平里程在原始航点高度之间线性插值，保持原航线高度剖面，而不是为了避障新增爬升或下降。
