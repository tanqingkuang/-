# 航线文件设计

## 目标

`route_file` / `rally_route_file` 是面向客户场景的航线输入边界。内部算法、仿真控制和避障后端仍只消费展开后的 `config["route"]` / `config["rally_route"]`，不直接感知外部航线文件格式。

## 设计原则

- 主配置只声明 `route_file` 或 `rally_route_file` 相对路径。
- 相对路径统一按主配置文件所在目录解析，不依赖当前工作目录。
- 航线文件策略只负责“读/写某种文件格式”，不做仿真语义解释。
- `LineFileManager` 负责路径解析和策略调度。
- `LineFileStrategyFactory` 负责选择具体策略；后续客户格式只新增策略并注册到工厂。

## 当前策略

`JsonLineFileStrategy` 支持 `.json` 文件，文件根对象就是原 `base.json.route` 或 `rally_demo.json.rally_route` 的完整结构。对外基础航线优先使用经纬高：

```json
{
  "speed_mps": 20.0,
  "waypoints": [
    {"latitude_deg": 39.0, "longitude_deg": 116.0, "altitude_m": 1000.0, "R": 0.0}
  ]
}
```

加载时以上述 `waypoints[0]` 作为 ENU origin。该 origin 不写入主配置文件，仅存在于加载后的配置副本中，供障碍经纬度转换和避障航线输出复用。高度不参与坐标转换，`altitude_m` 原样进入内部高度字段。

旧 ENU 字段 `x_m/y_m/altitude_m` 仍兼容，用于历史演示和测试夹具。避障航线输出在存在 origin 时输出经纬高；若输入配置没有经纬 origin，则保持旧 ENU 输出。

避障航线输出可能包含已烘焙圆弧航段，此时航点会额外带 `turn_sign` 与经纬度 `center`：

```json
{
  "latitude_deg": 39.001,
  "longitude_deg": 116.002,
  "altitude_m": 1000.0,
  "R": 0.0,
  "turn_sign": 1.0,
  "center": {"latitude_deg": 39.002, "longitude_deg": 116.002, "altitude_m": 1000.0}
}
```

控制器读取 `route.waypoints[]` 时兼容这些字段：`turn_sign` 为 `0` 或缺省时仍按普通折线和 `R` 处理，非 `0` 时按已知圆弧航段处理。

## 扩展方式

新增客户格式时：

1. 新增一个实现 `LineFileStrategy` 的策略类。
2. 在工厂注册该策略。
3. 策略的 `load()` 返回兼容 `config["route"]` / `config["rally_route"]` 的对象。
4. 策略的 `save()` 接收同一结构并输出客户需要的文件格式。

这样控制器、GUI 和算法层不需要跟随客户文件格式变化。
